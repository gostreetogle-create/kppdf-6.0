## 0. Контекст

Склад — это операционная точка контроля движения товара: что пришло, что ушло, что списали, что нужно закупить. Главный пользователь — кладовщик (`storekeeper`). Содержит **4 НОВЫЕ сущности v6** (`SupplierDelivery` СД-XXXX / `Shipment` ОТК-XXXX / `WriteOffAct` АС-XXXX / `PurchaseOrder` ЗП-XXXX) + справочники `StockRecord` / `StockMovement` / `Reservation` / `PurchaseRequest`. Ключевое правило — `StockMovement immutable` (никогда не редактируется задним числом, только создаётся встречное).

5 папок, но `02-*` расщеплена на **5 воркспейс-папок** (по URL: `/warehouse` `/supplier-deliveries` `/shipments` `/purchase` `/write-off`) — Стратегия C (URL/workspace) для multi-entity модуля.
