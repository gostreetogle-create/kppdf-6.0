# 04_Склад/00-spr/00-glossary.md — Глоссарий терминов модуля «Склад»

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §4](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение содержанием — Аналитик Run 4 (ТЗ-009).

## Назначение

Глоссарий терминов модуля `Склад` — единый источник для всех downstream-документов. Все термины должны совпадать с [`GLOSSARY-MASTER.md`](../../99_Справочники/GLOSSARY-MASTER.md).

## Термины (12 ключевых, из ПОДРОБНЫЙ.md §0 + UI.md §0)

| Термин | Что значит |
|---|---|
| **Warehouse (Склад)** | Физическое место хранения. Справочник: «Основной», «У партнёра», «Транзитный». |
| **StockRecord (Остаток)** | Текущее количество товара на складе: `quantity` (физически) + `reservedQuantity` (под КП/ЗК). |
| **StockMovement (Движение)** | Immutable запись: приход/расход/transfer/write_off. Нельзя править задним числом. |
| **SupplierDelivery (Приход)** | Документ приёмки товара от поставщика. Номер **СД-XXXX**. |
| **Shipment (Отгрузка)** | Документ отгрузки клиенту. Номер **ОТК-XXXX**. |
| **WriteOffAct (Акт списания)** | Документ списания брака/недостачи. Номер **АС-XXXX**. Требует approve. |
| **PurchaseOrder (Закупка)** | Внутренний документ «купить N штук у поставщика». Номер **ЗП-XXXX**. |
| **PurchaseRequest (Заявка от системы)** | Авто-заявка при `availableQty < minStock`. Закупщик решает: оформить ЗП или отложить. |
| **Supplier (Поставщик)** | `Organization` с ролью «поставщик» (`isSupplier=true`). |
| **availableQty (Доступно)** | = `quantity − reservedQuantity`. Цифра для менеджера по продажам. |
| **minStock (Минимум)** | Минимальный остаток. При падении ниже — авто-триггер PurchaseRequest. |
| **Snapshot (Снимок)** | Копия данных о товаре (SKU, название, цена) на момент операции. |

> **Префиксы документов:** СД-XXXX (SupplierDelivery), ОТК-XXXX (Shipment), АС-XXXX (WriteOffAct), ЗП-XXXX (PurchaseOrder). Каждый — отдельный sealed counter (СПОР-13).

## Принципы наполнения (Аналитик Run 4)

1. **Один термин — одно определение.** Если пересекается с КП/Производством/Договором — дать cross-ref, не дублировать.
2. **Два источника:** ПОДРОБНЫЙ.md §0 (модельные термины) + UI.md §0 (пользовательские термины). Объединить, без дублирования.
3. **4 НОВЫХ сущности** (PurchaseOrder/SupplierDelivery/Shipment/WriteOffAct) — GAP №8 закрыт.
