# 04_Склад/00-spr/00-clients.md — Клиенты модуля «Склад»

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §4](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение содержанием — Аналитик Run 4 (ТЗ-009).

## Назначение

Кому принадлежат остатки на складе. В `Shipment` — поле `customerId` (Organization), в `SupplierDelivery` — опциональная привязка к клиенту через `parentContractId → Contract → customerId`. Для v1 клиенты — это те же `Organization` из общего справочника.

## Содержимое (планируется)

- **Shipment.customerId** — обязательный FK → Organization. Клиент, которому отгружаем товар.
- **SupplierDelivery.parentContractId** — опциональный FK → Contract. Если приход по тендерному Договору — привязка к клиенту через Договор.
- **Поставщики ≠ клиенты** — поставщик (isSupplier=true) и клиент — могут быть разными организациями.
