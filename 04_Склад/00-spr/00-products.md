# 04_Склад/00-spr/00-products.md — Товары модуля «Склад»

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §4](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение содержанием — Аналитик Run 4 (ТЗ-009).

## Назначение

SKU-система, категории товаров, minStock (минимальный остаток), единицы измерения. Товары — общий справочник (`Product`), но для Склада важны поля `minStock` и `isActive`.

## Содержимое (планируется)

### SKU-система

- Каждый товар имеет `sku` (уникальный артикул) и `name` (название).
- Категории товаров — справочник (для фильтрации в `/warehouse`).
- Единицы измерения: шт / комплект / м² / м.п. / кг / час.

### minStock

- Поле `Product.minStock` — минимум товара на складе.
- Фоновый job ежедневно проверяет `StockRecord.availableQty < Product.minStock` → авто-создание `PurchaseRequest.reason = low_stock`.

### Snapshot-поля

При каждой операции (СД/ОТК/АС/ЗП) создаётся **снимок** товара: `productSku`, `productName`, `unit`. Это защищает от изменений в справочнике товаров.
