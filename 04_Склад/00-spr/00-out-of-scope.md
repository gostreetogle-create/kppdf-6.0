# 04_Склад/00-spr/00-out-of-scope.md — Что НЕ входит в модуль «Склад»

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §4](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение содержанием — Аналитик Run 4 (ТЗ-009).

## Назначение

Явный список функций, которые НЕ входят в модуль Склад v1. Предотвращает scope creep при разработке.

## Что НЕ входит в v1 (из UI.md §8)

### UI не входит
- **Drag-and-drop** для строк в таблицах — порядок меняется стрелочками.
- **Мобильный интерфейс** для кладовщика (Android + сканер штрих-кодов) — v2.
- **Тёмная тема** — отдельный вопрос.
- **Многоязычность** — отдельный вопрос.
- **Real-time** через WebSocket (F5 или 30 сек обновление) — v2.

### Функции не входят
- **Штрих-коды** для сканирования — v2.
- **Статистика оборачиваемости** (turnover days) — v2.
- **ABC/XYZ анализ** — v2.
- **ML-прогнозирование спроса** — **никогда** (СТЕК-ПРЕДПИСАНИЕ §6.10).
- **Интеграция с 1С / МойСклад** — v2.
- **Multi-currency** — v2 (СПОР-14, в v1 только RUB).
- **CDN для фото товаров** — v2.
- **Аудит-лог изменений** (история правок) — v2.
- **Возврат от клиента** (R2 / S3) — v2. В v1 только WriteOffAct для внутреннего брака.
- **Шаблоны документов** с логотипом/печатью — v2 (в v1 только табличный view).
- **Прогнозирование спроса** (ML) — никогда.

### Out-of-scope для v1
- Принятие прихода без `PurchaseOrder` — допустимо (машина без предупреждения), но `supplierPurchaseOrderId = NULL`.
- Возврат от клиента через `Shipment.returned` — v2.
- Автоматическое создание `Shipment` при завершении ЗК — НЕ создаёт, только предложение.
