# 00-products.md — Сущность «Продукт»: Товары / Услуги / Работы

> **Назначение.** Сущность `Product` с типом через enum `kind: ITEM | SERVICE | WORK`. Поля: артикул (SKU), название, описание, категория, фото (загрузка с серверным сжатием JPEG max 1200×800 / WebP, **без кроппера в v1**), единица измерения, базовая цена (`basePrice`), себестоимость (`costPrice`, NULL допустим для проектных/сборных), архивный флаг `isActive`.

> ⚠️ **STUB** (создан Архитектором 2026-06-26, PSL-004). Будет заполнен Моделировщиком (поля + FK к категории) + Бизнес-аналитиком (правила для kind=ITEM/SERVICE/WORK и видимости себестоимости) в Phase 1 Bootstrap согласно [AGENT-ROLES.md §2.3 + §2.2](../../AGENT-ROLES.md). Источник: §4 «Карточка товара» + §6 колонка «Тип» исходного `01_КП/МОДУЛЬ-КОММЕРЧЕСКОЕ-ПРЕДЛОЖЕНИЕ.md` (распущен в этом коммите). После начала работы **этот STUB не трогать** (Правило 5.2.2 [AGENT-METHOD.md](../../AGENT-METHOD.md)).

## 0. Контекст

Каталог товаров — общий для всех документов (КП, Договор как Спецификация, ЗК как позиции). Видимость полей управляется через GAP-014 (см. `04-rbac.md`): `costPrice` виден только admin/director/accountant. Сменная наценка по категории товара — **задел на будущее** (поле моделируется сейчас, чтобы не переделывать БД в v2).
