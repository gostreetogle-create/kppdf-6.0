# 00-clients.md — Сущность «Клиенты»: Юрлица / ИП / Физлица

> **Назначение.** Сущность `Client` с тремя категориями контрагентов: Юрлица (ООО/АО/ЗАО — полные реквизиты), ИП (ИНН + ОГРНИП, без КПП), Физлица (без ИНН/КПП/ОГРН, только ФИО+адрес+контакт). Контактные лица (множественные). Проверка дублей при создании: для юрл. и ИП — по ИНН; для физл. — по ФИО + дата рождения.

> ⚠️ **STUB** (создан Архитектором 2026-06-26, PSL-004). Будет заполнен Моделировщиком (поля + категории через `ClientType` enum) + Бизнес-аналитиком (правила подстановки и дублей) в Phase 1 Bootstrap согласно [AGENT-ROLES.md §2.3 + §2.2](../../AGENT-ROLES.md). Источник: §5 dropdown «Клиент» исходного `01_КП/МОДУЛЬ-КОММЕРЧЕСКОЕ-ПРЕДЛОЖЕНИЕ.md` (распущен в этом коммите). После начала работы **этот STUB не трогать** (Правило 5.2.2 [AGENT-METHOD.md](../../AGENT-METHOD.md)).

## 0. Контекст

Это «их сторона» в цикле КП. Универсальное правило §9: пустые реквизиты в footer НЕ выводятся (без прочерков «—»). Иностранный клиент без БИК → строка «БИК» отсутствует в footer. Физлицо — отдельная полноценная категория, не «исключение для клиента без ИНН».
