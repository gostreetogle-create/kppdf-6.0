# 01-shablon/ — Шаблоны документов (скелеты для будущих КП)

> **Назначение.** Папка для визуальных макетов КП (DocumentTemplate + блоки: text/table/separator/header/image/footer/totals + токены {{...}}).
>
> **Контекст.** Раздел «Конструктор шаблонов» и «Готовые текстовые блоки» в исходном `01_КП/МОДУЛЬ-КОММЕРЧЕСКОЕ-ПРЕДЛОЖЕНИЕ.md` — кандидаты на декомпозицию в эту папку.
>
> **Создано как:** ⚠️ STUB (`§4.5 AGENT-METHOD.md Правило 4.5.1`). Содержимое будет наполнено Архитектором и Бизнес-аналитиком по их зонам ответственности.

---

## 0. Контекст

Это первая папка после фундамента (`00-spr/`). Представляет «скелеты» документов: админ собирает шаблон, КП потом наполняется по этому шаблону.

## 1. Ожидаемое содержимое (для будущего Архитектора)

- `01-konstruktor.md` — конструктор шаблонов: блоки, токены, drag-and-drop (v1 — без), фиксированный порядок блоков.
- `01-primery.md` — 4 hardcoded шаблона v1 (чтобы КП работал «из коробки»).

## 2. Связанные документы

- [`../../AGENT-ENTRYPOINT.md`](../../AGENT-ENTRYPOINT.md) — общая точка входа
- [`../../AGENT-ROLES.md` §2.1](../../AGENT-ROLES.md) — границы роли Архитектора (структура, не контент)
- [`../../AGENT-METHOD.md` §4.5 Правило 4.5.1](../../AGENT-METHOD.md) — правила создания STUB-файлов
- [`../00-spr/00-otkrytye-voprosy.md`](../00-spr/00-otkrytye-voprosy.md) — локальные дыры модуля КП

## ⚠️ Статус STUB

Создан агентом (Parent) как pre-launch scaffolding для первого запуска Архитектора. **НЕ претендует на содержимое** — это рабочий слот, который будет заполнен ролями Architect (структура файлов) и BusinessAnalyst (бизнес-правила шаблонов).
