# 02-avtosoxranenie.md — Автосохранение черновика КП

> **Назначение.** Спецификация автосохранения: событийный debounce **5-10 секунд** после последнего изменения (НЕ по cron, таймер сбрасывается при каждом новом изменении); параллельная запись в localStorage **синхронно** при каждом изменении (защита от refresh); индикатор в правом нижнем углу (серый кружок → ✓ Сохранено ЧЧ:ММ / ⏳ Сохраняется... / ⚠ Не сохранено). Optimistic locking через `updatedAt` при параллельном редактировании двумя менеджерами.

> ⚠️ **STUB** (создан Архитектором 2026-06-26, PSL-004). Будет заполнен Моделировщиком (`updatedAt` поле + endpoint для сохранения) + Бизнес-аналитиком (правила debounce + восстановление из localStorage) в Phase 1 Bootstrap согласно [AGENT-ROLES.md §2.3 + §2.2](../../AGENT-ROLES.md). Источник: §8 «Автосохранение черновика» исходного `01_КП/МОДУЛЬ-КОММЕРЧЕСКОЕ-ПРЕДЛОЖЕНИЕ.md` (распущен в этом коммите). После начала работы **этот STUB не трогать** (Правило 5.2.2 [AGENT-METHOD.md](../../AGENT-METHOD.md)).

## 0. Контекст

Ограничение: восстановление из localStorage работает только в том же браузере/устройстве. Доехал до дома на ноуте — увидит последнюю серверную версию (ту, что попала в 5–10 сек debounce или по Ctrl+S). Зелёная галочка «✓ Сохранено ЧЧ:ММ» важна: менеджер должен убедиться, что последние правки дошли до сервера, прежде чем уходить с рабочего места.
