# 02-smena-yurlitsa.md — Killer-feature «30 секунд»

> **Назначение.** Спецификация двух способов смены организации-продавца в КП: (1) dropdown «Продавец» внутри существующего черновика — доступен только в статусе DRAFT; (2) кнопка «Скопировать для другого юрлица» — создаёт новый документ-черновик, доступна в любом статусе кроме Отклонено и Конвертировано. Логика «копируем и вставляем именно, потом редактируем»: авто-применение базовой наценки и ставки НДС нового юрлица, пересчёт цены продажи, жёлтое ⚠-предупреждение.

> ⚠️ **STUB** (создан Архитектором 2026-06-26, PSL-004). Будет заполнен UX-проектировщиком (состояния кнопки/dropdown, ⚠-предупреждение) + Бизнес-аналитиком (правила применения наценки/НДС по юрлицу, линейное версионирование) в Phase 1 Bootstrap согласно [AGENT-ROLES.md §2.4 + §2.2](../../AGENT-ROLES.md). Источник: §3 «30 секунд» целиком + §5 dropdown «Продавец» + §7 кнопка «Скопировать» + §9 правила смены организации исходного `01_КП/МОДУЛЬ-КОММЕРЧЕСКОЕ-ПРЕДЛОЖЕНИЕ.md` (распущен в этом коммите). После начала работы **этот STUB не трогать** (Правило 5.2.2 [AGENT-METHOD.md](../../AGENT-METHOD.md)).

## 0. Контекст

Это самая важная UX-цель v6 («15 мин → 30 сек»). Структура версий — **линейная** (не древовидная): каждая новая версия создаётся от текущего документа, цепочка восстанавливается через `parentProposalId`. Товары и базовые цены из каталога НЕ меняются при смене юрлица — меняется итоговая через наценку.
