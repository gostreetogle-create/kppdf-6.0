# 04-rbac.md — RBAC-матрица модуля КП

> **Назначение.** Кто из 7 ролей видит / редактирует / совершает какие действия над сущностями КП (Proposal, ProposalItem, Organization, Product). Расширяет сводную [`RBAC-MATRIX.md`](../../99_Справочники/RBAC-MATRIX.md) применительно к КП.
> **Автор.** Бизнес-аналитик (Run 1/5, ТЗ-002). Заполнен 2026-06-27.

## 0. Контекст

Каждое правило имеет ID формата `RBAC-КП-{TYPE}-{NNN}`, где TYPE ∈ {F (field visibility), A (action), OW (ownership), V (visibility filter), C (conditional)}.

Роли: `admin`, `director`, `accountant`, `manager`, `production`, `storekeeper`, `viewer`.

Обозначения: ✅ = разрешено, ❌ = запрещено, ⚠️ = с ограничением, 🔒 = с 2FA.

## 3. Ownership «свой» (OW-rules)

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| RBAC-КП-OW-001 | manager редактирует только КП где `createdById == user.id` | 403 Forbidden + скрыть в списке | RBAC-MATRIX §2.1 |
| RBAC-КП-OW-002 | manager видит в списке только свои + «доступные» (ACL для director/admin) | фильтр на API route `/proposals?filter=own` | RBAC-MATRIX §2.1 |
| RBAC-КП-OW-003 | Расторжение/архивирование КП — только creator или admin/director | 403 Forbidden | RBAC-MATRIX §1.1 + INV-КП-SOFT-002 |
| RBAC-КП-OW-004 | Восстановление из архива — только admin (даже creator не может) | «обратитесь к администратору» | RBAC §1.1 |
| RBAC-КП-OW-005 | Комментарии к КП — только creator/director/admin | 403 Forbidden | GAP-013 |

**Итого OW-rules: 5 правил.**

## 4. Visibility filters (V-rules)

| ID | Правило | Источник |
|---|---|---|
| RBAC-КП-V-001 | КП в статусах PAID/CONVERTED/ARCHIVED — режим «только чтение» для всех (даже creator) | МОДУЛЬ §2 + RBAC-КП-A-003 |
| RBAC-КП-V-002 | КП в статусе NEW_VERSION — видим только creator + admin + director | МОДУЛЬ §3 |
| RBAC-КП-V-003 | КП с `isArchived=true` не появляется в основном списке, только по фильтру «Архив» | INV-КП-SOFT-001 |
| RBAC-КП-V-004 | КП в статусе CANCELLED — редактирование заблокировано, доступны: просмотр + архивирование | МОДУЛЬ §3 |
| RBAC-КП-V-005 | КП в статусе SENT — редактирование заблокировано (только отмена) | МОДУЛЬ §3 |
| RBAC-КП-V-006 | КП в статусе ACCEPTED — редактирование заблокировано (только конвертация / оплата) | МОДУЛЬ §3 |
| RBAC-КП-V-007 | PDF КП **никогда** не содержит себестоимость (даже для admin/director) | GAP-014 + INV-КП-MISC-002 |

**Итого V-rules: 7 правил.**
