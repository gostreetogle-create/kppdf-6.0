# 04-biznes-pravila.md — Бизнес-инварианты модуля КП

> **Назначение.** Канонические правила модуля КП, которые должны выполняться ВСЕГДА. Нарушение = баг.
> **Автор.** Бизнес-аналитик (Run 1/5, ТЗ-002). Заполнен 2026-06-27.

## 0. Контекст

Каждое правило имеет ID `INV-КП-{GROUP}-{NNN}` где GROUP ∈ {POS, PRICE, DISC, VAT, CONV, PAY, SIGN, REQ, SOFT, MISC}.

---

## 1. Группа POS — Позиции

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-POS-001 | КП должен иметь ≥ 1 позицию | «Добавьте хотя бы один товар», кнопка «Создать КП» disabled | МОДУЛЬ §4 |
| INV-КП-POS-002 | Каждая позиция должна иметь `productId` (FK на Product) | «Выберите товар», подсветка строки | МОДУЛЬ §4 |
| INV-КП-POS-003 | Удаление последней позиции запрещено | Кнопка ⌫ в последней строке disabled | МОДУЛЬ §4 |
| INV-КП-POS-004 | Дублирование товара в КП запрещено (один product — одна строка) | «Этот товар уже добавлен в КП» | МОДУЛЬ §4 |

---

## 2. Группа PRICE — Цены

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-PRICE-001 | `price ≥ 0` | red border + «Цена не может быть отрицательной» | МОДУЛЬ §4 |
| INV-КП-PRICE-002 | Итого по позиции = `quantity × price × (1 + markup) × (1 - discount)`, авто-расчёт | При ручной правке итога → пересчёт скидки | МОДУЛЬ §6 + OQ-003 |
| INV-КП-PRICE-003 | Если итоговый price < sum(costPrice × quantity) → ⚠ продажи ниже себестоимости | ⚠ возле итоговой суммы (жёлтая для manager, красная для director) | МОДУЛЬ §9 |
| INV-КП-PRICE-004 | Цена не округляется — хранить с точностью 0.01 ₽ (Decimal(18,2) или Int в копейках) | В БД Decimal(18,2) | SCHEMA-CONSOLIDATED §3.4 |
| INV-КП-PRICE-005 | Базовая цена берётся из каталога товаров (`Product.basePrice`) и фиксируется в snapshot при добавлении | Не пересчитывается при изменении каталога | SCHEMA-CONSOLIDATED §0 принцип 3 |

---

## 3. Группа DISC — Скидки

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-DISC-001 | `discount ∈ [0, 100]` (%) | «Скидка от 0 до 100%» | МОДУЛЬ §4 |
| INV-КП-DISC-002 | Маржа (margin) может быть отрицательной (продажа ниже себестоимости) — это НЕ ошибка | См. INV-КП-PRICE-003 (предупреждение, не блок) | МОДУЛЬ §9 |
| INV-КП-DISC-003 | Скидка применяется ПОСЛЕ `quantity × price × (1 + markup)`, ДО НДС | Формула: price × qty × (1 + markup) × (1 - discount) | МОДУЛЬ §4 + OQ-003 |
| INV-КП-DISC-004 | Общая скидка КП применяется ко всем позициям; индивидуальная в ячейке перекрывает общую | Пометка «общ.» в скрытом столбце | МОДУЛЬ §6 |

---

## 4. Группа VAT — НДС

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-VAT-001 | Один КП имеет ОДНУ ставку НДС на все позиции | При попытке смешать → красная подсветка + «выберите одну ставку» | СПОР-15 + МОДУЛЬ §5 |
| INV-КП-VAT-002 | Допустимые значения `vatRate`: 0 (без НДС), 5%, 10%, 20% или customNumber | «Неизвестная ставка НДС» + dropdown | МОДУЛЬ §4 |
| INV-КП-VAT-003 | Дефолт `vatRate` — по последнему КП менеджера (per-user sticky) | Применяется автоматически; manager может изменить | UX-принцип 2 (Smart Suggestions) |

---

## 5. Группа CONV — Конвертация в Договор

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-CONV-001 | **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО** конвертировать PAID КП в Договор | Кнопка «Конвертировать» скрыта; API 409 Conflict | СПОР-5 + INV-КП-CONV-001 |
| INV-КП-CONV-002 | Конвертация создаёт Договор со снапшотом цен (`ContractItem.priceWithoutVat`) | Снимок в момент конвертации | SCHEMA-CONSOLIDATED §0 принцип 3 |
| INV-КП-CONV-003 | `parentProposalId` на Contract = NOT NULL (Договор всегда от КП) | API 400 «Договор может быть создан только из КП» | МОДУЛЬ §3 + SCHEMA-CONSOLIDATED §6.1 |
| INV-КП-CONV-004 | Один КП = один Договор. Повторная конвертация → ошибка | «Договор уже создан для этого КП» | МОДУЛЬ §2 |

---

## 6. Группа PAY — Оплата

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-PAY-001 | «Оплачено» означает зарегистрирован входящий Payment в Finance для этого КП | Кнопка «Отметить оплачено» доступна только creator/director/admin | RBAC-MATRIX §1.1 + СПОР-4 |
| INV-КП-PAY-002 | После PAID авто-создаётся ProductionOrder (ЗК) со статусом CREATED | Триггер в FLOW-MAP §2 | FLOW-MAP + МОДУЛЬ §8 |
| INV-КП-PAY-003 | PAID → CONVERTED запрещён (см. INV-КП-CONV-001) | Кнопка скрыта | СПОР-5 |
| INV-КП-PAY-004 | При ошибочном PAID — только через бухгалтера (Refund/STORNO + ручное закрытие Order) | Авто-отката нет в v1 | FLOW-MAP §3.2 |

---

## 7. Группа SIGN — Подписанты

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-SIGN-001 | Подписант обязателен для юрлица обеих сторон | «Заполните реквизиты подписанта» | МОДУЛЬ §4 |
| INV-КП-SIGN-002 | Доверенность (`attorney`) — если подписант ≠ руководитель по ЕГРЮЛ | Warn если нет доверенности, НЕ блок | МОДУЛЬ §4 |
| INV-КП-SIGN-003 | Логотип / печать НЕ загружаются менеджером; только admin/director с 2FA | Кнопка hidden, см. RBAC-MATRIX §2.3 | RBAC-MATRIX §2.3 + GAP-016 |

---

## 8. Группа REQ — Реквизиты

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-REQ-001 | Пустые реквизиты НЕ выводятся в footer (без прочерков «—») | В footer: только заполненные поля | МОДУЛЬ §9 |
| INV-КП-REQ-002 | ИНН клиента обязателен для отправки КП | «Укажите ИНН клиента» | МОДУЛЬ §5 |
| INV-КП-REQ-003 | При смене юрлица после создания позиций — цены НЕ пересчитываются | Banner: «Цены не изменились» | МОДУЛЬ §3 |
| INV-КП-REQ-004 | Шаблон (`templateId`) обязателен для создания КП | Кнопка «Создать КП» disabled без шаблона | OQ-002 |

---

## 9. Группа SOFT — Soft-delete и архивирование

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-SOFT-001 | КП никогда не удаляется физически, только `isArchived=true` | Физический DELETE запрещён | INV-КП-SOFT-001 |
| INV-КП-SOFT-002 | Архивирование доступно creator + admin/director | 403 для accountant/viewer | RBAC-КП-A-007 |
| INV-КП-SOFT-003 | Разархивирование — только admin | Даже creator не может разархивировать; «обратитесь к администратору» | RBAC §1.1 |

---

## 10. Группа MISC — Прочее

| ID | Правило | Следствие при нарушении | Источник |
|---|---|---|---|
| INV-КП-MISC-001 | Auto-save каждые 5-10 сек + localStorage persist | Потеря данных при сбое браузера | МОДУЛЬ §8 |
| INV-КП-MISC-002 | Файл КП (PDF) **никогда** не содержит себестоимость (даже для director/admin в директорском режиме) | Утечка себестоимости клиенту | GAP-014, RBAC §3.1 |
| INV-КП-MISC-003 | Нумерация КП-YYYY-NNNN по `Counter` таблице (orgType='KP', year reset) | Конфликт счётчиков | SCHEMA-CONSOLIDATED §4 |
| INV-КП-MISC-004 | Режим `mode_director` (по умолчанию ВЫКЛ) превращает жёлтую ⚠ в красную подсветку | Если `mode_director=true` постоянно — мешает обычной работе | МОДУЛЬ §9 |
| INV-КП-MISC-005 | Optimistic locking через `updatedAt` при параллельном редактировании | При конфликте: warning «Кто-то другой редактировал» | СПОР-6 + МОДУЛЬ §8 |
| INV-КП-MISC-006 | Валюта жёстко RUB в v1 | multi-currency отложен в v2 | СПОР-14 |
