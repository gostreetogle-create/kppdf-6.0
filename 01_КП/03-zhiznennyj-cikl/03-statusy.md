# 03-statusy.md — 8 статусов КП + диаграмма переходов

> **Назначение.** State-машина КП: 8 статусов с определениями и разрешёнными переходами. `DRAFT` (черновик, новый или «Создать новую версию» → новый документ) / `NEW_VERSION` (промежуточное состояние создания версии) / `SENT` (отправлено клиенту) / `ACCEPTED` (принято клиентом) / `PAID` (оплачено → авто-создание ЗК) / `CONVERTED` (в Договор) / `ARCHIVED` (в архиве, видим по фильтру) / `CANCELLED` (отменено).

> ⚠️ **STUB** (создан Архитектором 2026-06-26, PSL-004). Будет заполнен Бизнес-аналитиком (определения статусов + ASCII-схема переходов) в Phase 1 Bootstrap согласно [AGENT-ROLES.md §2.2](../../AGENT-ROLES.md). Источник: §2 «Полная карта статусов» + ASCII-диаграмма + §14 «Запрет КП Оплачено → Договор» исходного `01_КП/МОДУЛЬ-КОММЕРЧЕСКОЕ-ПРЕДЛОЖЕНИЕ.md` (распущен в этом коммите). После начала работы **этот STUB не трогать** (Правило 5.2.2 [AGENT-METHOD.md](../../AGENT-METHOD.md)).

## 0. Контекст

Формат номера КП-XXXX (по порядку). КРИТИЧНОЕ СОГЛАСОВАНИЕ: КП в статусе `PAID` **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО** конвертировать в Договор (правовая логика: КП не считается офертой после оплаты — требуется подписанный Договор). Цепочка: ACCEPTED → Конвертация → подписание Договора → оплата. Согласовано PO 24.06.2026.
