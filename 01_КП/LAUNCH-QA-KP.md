6. **Развилка** по §5:
   - **Ветка А (🔴 P0):** вернуть Аналитику на исправление, создать новую сессию Codebuff
   - **Ветка B (✅ пройдено):** записать **PSL-NNN** (Pipeline v6 Run 1/5 QA PASSED) → commit + push → перейти к **Run 2/5 Аналитика**

## 3. Зоны особого внимания (Фокус Run 1)

QA должен прицельно проверить эти места — самые частые источники ошибок:

### 3.1 RBAC-матрица (`04-rbac.md`)

**Что проверять прицельно:**
- **48 ячеек** (6 ролей × 8 действий) заполнены все — нет пустых / `N/A` без обоснования.
- Менеджер **НЕ видит `costPrice`** (GAP-014 закрыт, источник СПОР-14: RUB-only) — проверить явно.
- Картотека-комментарии только для `director/admin` (GAP-013) — проверить.
- Логотип/печать только для `admin/director` с 2FA (GAP-016) — проверить.
- **SoD (разделение обязанностей):** одна роль не может и создать, и подтвердить (например, WriteOffAct создатель ≠ approver; Payment creator ≠ OrderClosing approver).

### 3.2 Инварианты (`04-biznes-pravila.md`)

**Что проверять:**
- 9 групп инвариантов покрыты:
  1. ≥1 позиции в корзине
  2. Цена ≥ 0
  3. Скидка ∈ [0, 100]
  4. Наценка свободно (но ≥ -100% по здравому смыслу)
  5. Налоги / VAT
  6. **Конвертация Оплачено → Договор 🛑** (СПОР-11 + правка B)
  7. Пустые реквизиты (Organization / Client / Product)
  8. ⚠️-иконка для просроченных (визуальное состояние, не правило)
  9. 1 ЗК из 1 PAY (СПОР-5 триггер)
- Каждое правило имеет **уникальное следствие** при нарушении (AGENT-ROLES.md §2.2: «У каждого правила есть уникальное следствие при нарушении»).
- Каждое правило имеет **явную ссылку на источник** (`_Источник: СПОР-N (24.06.2026)_` или `_Источник: OQ-NNN_`).

### 3.3 State-машина КП (`03-statusy.md`)

**Что проверять:**
- 8 статусов (DRAFT, SENT, ACCEPTED, REJECTED, EXPIRED?, PAID, CONVERTED, ARCHIVED) — каждый имеет **определение**, **переходы**, **доступные действия**.
- ASCII-схема переходов присутствует и читаема.
- **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО КП Оплачено → Договор** (явное правило + визуальное отражение в схеме).
- **Dead states** (`REJECTED`, `ARCHIVED`) обоснованы — допустимы.
- **Unreachable states** нет — каждый статус достижим минимум одним переходом.
- **Invalid transitions** (например, `DRAFT → PAID` минуя `SENT/ACCEPTED`) → ошибка с понятным сообщением.

### 3.4 Cross-doc drift (ВСЕ 3 файла Run 1)

- Терминология единая (от `99_Справочники/GLOSSARY-MASTER.md`).
- Нет дублирования информации между 3 файлами (каждое правило = в одном месте).
- Версии обновлены (header каждого файла: 0.1 → 0.2).
- Кросс-ссылки валидны ([§5 Связанные документы](#) работают).

✅ **CHECK 3: Приоритеты расставлены.** Все дыры ранжированы 🔴 P0 / 🟡 P1 / 🟢 P2. Без приоритета = невалидный отчёт.

✅ **CHECK 4: Табличная форма соблюдена.** §4.1 + §4.2 + §4.3 — все в таблицах. Свободный нарратив = нарушение анти-паттерна A3 (AGENT-FORMAT.md).
