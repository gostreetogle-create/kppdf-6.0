## 0. Контекст

### 0.1 Что такое Karkas-Kit

Karkas-Kit = единый UI-патерн проекта KPPDF CRM v6. Название от слова «каркас» (frame/skeleton) — набор стандартных UI-компонентов и поведений, единых для ВСЕГО продукта. Концепция из [02-redaktor.md](02-konstruktor-kp/02-redaktor.md): «Сначала загружаются глобальные компоненты (3-зонный макет), затем встраивается локальная специфика (поля КП)».

### 0.2 Связь с принципами

4 канонических UX-принципа ([AGENT-METHOD.md §5.4](../../AGENT-METHOD.md#54)):

| # | Принцип | Кратко | Приоритет |
|---|---|---|---|
| 1 | **Единообразие** (Consistency) | Все элементы ведут себя одинаково во всём продукте | Самый высокий |
| 2 | **Интеллектуальные подсказки** (Smart Suggestions) | Система активно помогает, а не пассивно ждёт | Высокий |
| 3 | **Логичная последовательность** (Intuitive Flow) | Следующее действие очевидно из текущего состояния | Средний |
| 4 | **Минимизация трения** (Low Friction) | Минимум кликов для типовых сценариев | Самый низкий |

Приоритет при конфликте: (1) > (2) > (3) > (4) — Правило 5.3.4.

### 0.3 Связь с RBAC и state-машиной

Каждый компонент имеет секции:
- **RBAC-видимость**: кто может видеть/использовать (по [RBAC-MATRIX.md](../RBAC-MATRIX.md)).
- **State-зависимость**: в каких статусах КП доступен (по [03-statusy.md](03-zhiznennyj-cikl/03-statusy.md)).

| Зона | Положение | Контент | Ширина |
|---|---|---|---|

### 1.2. Поведение (4 UX-принципа)

| Принцип | Применение |
|---|---|
| (1) Единообразие | Этот 3-зонный макет — единый паттерн для ВСЕХ документов модуля КП. Договор / ЗК получат аналогичный, но адаптированный (3-зонный или 2-зонный). |
| (2) Подсказки | В центральной зоне над каждой позицией таблицы — tooltip с подсказкой формата. |
| (3) Последовательность | Левая зона — dropdowns (что заполняем) → Центральная (что получается) → Правая (что с этим делать). |
| (4) Минимизация трения | Resize-ручки между зонами (draggable splitter). По умолчанию split 20%/60%/20%. |

## 2. Button — primary action

### 2.1. Каноническая форма

| Атрибут | Значение | Источник (принцип) |
|---|---|---|
| Положение | Правая зона (R), верхний правый угол формы | (1) Единообразие |
| Иконка | 💾 (дискета) слева от текста | (1) |
| Текст | «Сохранить» (НЕ «OK», НЕ «Готово», НЕ «Применить») | (1) |
| Цвет | primary (Mantine `color="blue"`) | TODO: UI-VISUAL-PRINCIPLES.md |
| Размер | `size="md"` (40px высота) | (1) |
| Shortcut | `Ctrl+S` / `Cmd+S` | (4) Минимизация трения |
| Hover | subtle background darken + tooltip «Сохранить (Ctrl+S)» | (3) Последовательность |
| Focus | синяя рамка 2px + auto-focus on tab | (1) |
| Disabled | серый + cursor not-allowed + tooltip «Заполните обязательные поля» | (2) Подсказки |
| Loading | spinner внутри кнопки, текст меняется на «Сохранение...» | (1) |
| Success (после клика) | toast «КП сохранён» внизу справа на 3 сек | (1) |
| Error (при ошибке) | toast (danger) + форма остаётся редактируемой | (1) |

### 2.2. Применение в модуле КП

- Главная кнопка в редакторе КП (3-зонный макет §1).
- В модалке «Создать новую версию» (подтверждение).
- В диалоге «Подтвердить удаление» — это danger-вариант (§3), НЕ primary.

### 2.3. RBAC

| Действие | admin | director | accountant | manager | production | storekeeper | viewer | Источник |
|---|:-:|:-:|:-:|:-:|:-:|:-:|:-:|---|
| Save КП | ✅ | ✅ | ❌ | ✅ свой | ❌ | ❌ | ❌ | RBAC-КП-A-003 |

### 2.4. State-зависимость

- Активна в DRAFT и NEW_VERSION.
- Скрыта в SENT, ACCEPTED, PAID, CONVERTED, ARCHIVED.
- В CANCELLED — активна только если редактируется (обычно нет).

### 2.5. Danger-вариант

| Атрибут | Значение |
|---|---|
| Цвет | danger (красный) |
| Текст | «Удалить» / «Расторгнуть» |
| Триггер | только destructive actions |
| Shortcut | Enter (не Escape!) |
| Focus | НЕ autofocus (защита от случайного клика) |

## 4. Modal — диалог подтверждения действия

### 4.1. Каноническая форма

| Атрибут | Значение | Принцип |
|---|---|---|
| Trigger | destructive action (Удалить / Расторгнуть / Архивировать) | — |
| Header | Заголовок действия («Удалить КП?») + крестик справа (Esc) | (1) |
| Body | Краткое описание последствий («Это действие нельзя отменить. КП будет помечен как архивный.») | (3) |
| Footer | Слева — Cancel (вторичная), справа — Confirm (danger, если destructive) | (1) |
| Cancel button | «Отмена», серый outline, Esc-shortcut | (1) |
| Confirm button | красный danger («Удалить»/«Расторгнуть»), Enter-shortcut | (1) |
| Backdrop | затемнение 50% + click-outside закрывает ТОЛЬКО если не destructive | (2) |
| Анимация | fade-in 150ms, scale 0.95 → 1.0 | (3) |
| Фокус | autofocus на Cancel (НЕ на Confirm — защита от случайного клика) | (4) |

### 4.2. Где применяется в КП

- Подтверждение удаления позиции из корзины.
- Подтверждение расторжения КП.
- Подтверждение «Конвертировать в Договор» (НЕ destructive, но требует явного подтверждения).
- Подтверждение «Отметить оплачено».

### 4.3. RBAC

Cancel — доступен всем, кто видит trigger. Confirm — по RBAC-таблице действия (по RBAC-КП-A-NNN).

### 4.4. State-зависимость

Модалка открывается только из статусов, где соответствующая кнопка доступна (см. §1.4).

## 5. Toast — короткие уведомления

### 5.1. Каноническая форма

| Тип | Цвет | Иконка | Пример | Auto-close |
|---|---|---|---|---|
| success | зелёный | ✓ | «КП сохранён» | 3 сек |
| info | синий | ℹ️ | «Черновик восстановлен из локального хранилища» | 5 сек |
| warning | жёлтый | ⚠ | «Срок действия КП истекает через 3 дня» | НЕ auto-close (важно) |
| danger / error | красный | ✕ | «Ошибка сохранения. Попробуйте ещё раз.» | НЕ auto-close (action required) |

### 5.2. Положение

Всегда внизу справа. НЕ блокирует UI. Можно закрыть кликом.

### 5.3. Поведение

- Стек до 5 toast'ов; новые вытесняют старые.
- Hover — пауза auto-close.
- `Shift+Click` на toast — копировать текст в буфер (полезно для error trace).

### 5.4. Примеры в КП

| Событие | Тип toast | Текст |
|---|---|---|
| Успешное сохранение | success | «КП сохранён» |
| Восстановление из localStorage | info | «Черновик восстановлен из локального хранилища» |
| Истекающий срок | warning | «Срок действия КП истекает через 3 дня» |
| Ошибка сервера | danger | «Ошибка сохранения. Попробуйте ещё раз.» |
| Optimistic lock conflict | warning | «Документ был изменён другим пользователем. Обновить?» |

## 8. DateInput — дата

### 8.1. Каноническая форма

- Стандартный Mantine `DateInput`.
- Локаль: ru (`dd.mm.yyyy`).
- Min: сегодня (для КП — нельзя создать КП с прошедшей датой).
- Default: today + validity period (например, today + 14 дней).

### 8.2. Где применяется в КП

| Поле | Настройки | Источник |
|---|---|---|
| Дата КП | default = today | [02-redaktor.md](02-konstruktor-kp/02-redaktor.md) |
| Срок действия (validityPeriod) | default = today + 14 дней | UX-принцип 4 |

### 8.3. Edge-case: истёкший срок

При попытке действия с КП, у которого `validUntil < today` — мягкая ⚠-блокировка с подтверждением (из [04-biznes-pravila.md](04-pravila/04-biznes-pravila.md)).

## 9. Tooltip — подсказка при наведении

### 9.1. Каноническая форма

| Атрибут | Значение |
|---|---|
| Trigger | hover (≥ 300ms) или focus |
| Положение | auto (top если есть место, иначе bottom, иначе right, иначе left) |
| Задержка появления | 300ms |
| Скрытие | 100ms после mouseleave |
| Контент | plain text или с иконкой ℹ️ |
| Multi-line | поддерживается (\n) |
| Rich content | НЕ поддерживается в v1 (только текст) |

### 9.2. Где применяется

- Иконка ❓ возле сложного поля → tooltip с форматом и примером.

## 13. Banner / Alert — предупреждение зоны

### 13.1. Каноническая форма

| danger | красный | ✕ | «Продажа ниже себестоимости» (режим руководителя) |

- Размещается вверху зоны (над контентом).
- Закрытие: крестик справа (для info/warning); НЕ закрывается (для danger).
- Не блокирует взаимодействие с контентом ниже.

### 13.3. Применение в КП

|---|---|---|
| Истекающий срок | warning | «Срок действия КП истекает через 3 дня» |

## 23. Индикатор автосохранения

### 23.1. Каноническая форма (из [02-avtosoxranenie.md](02-konstruktor-kp/02-avtosoxranenie.md))

| Состояние | Индикатор | Положение |
|---|---|---|
| Сохранено | ✓ Сохранено ЧЧ:ММ (серый кружок) | правый нижний угол |
| Сохраняется | ⏳ Сохраняется... (серый кружок) | правый нижний угол |
| Ошибка | ⚠ Не сохранено (красный кружок) | правый нижний угол |

### 23.2. Поведение

- Debounce 5-10 сек после последнего изменения (таймер сбрасывается при каждом изменении).
- Параллельная запись в localStorage синхронно при каждом изменении (защита от refresh).
- Optimistic locking через `updatedAt` при параллельном редактировании.
