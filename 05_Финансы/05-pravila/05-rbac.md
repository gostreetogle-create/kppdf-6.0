## Назначение

Документирует **RBAC матрицу** модуля «Финансы»: какая роль может выполнять какое действие. Источник: `МОДУЛЬ-ФИНАНСЫ.md` §6 RBAC-таблица.

## 4 роли × 10 действий (≥30 правил)

| Действие | Менеджер | Бухгалтер | Директор | Админ |
|---|---|---|---|---|
| **Видеть свой список Order** | ✅ свои | ✅ всех | ✅ всех | ✅ всех |
| **Видеть карточку Order** | ✅ свой | ✅ всех | ✅ всех | ✅ всех |
| **Создавать Invoice (счёт)** | ❌ | ✅ | ✅ | ✅ |
| **Редактировать Invoice (DRAFT)** | ❌ | ✅ | ✅ | ✅ |
| **Выставить Invoice (DRAFT → ISSUED)** | ❌ | ✅ | ✅ | ✅ |
| **Регистрировать Payment (INCOMING)** | ❌ | ✅ | ✅ | ✅ |
| **Сторнировать Payment (STORNO)** | ❌ | ✅ | ✅ | ✅ |
| **Оформить Refund** | ❌ | ✅ | ✅ | ✅ |
| **Закрывать Order (AWAITING_PAYMENT → CLOSED)** | ❌ | ✅ (шаг 1) | ✅ (шаг 2) | ✅ |
| **Видеть маржинальность** | ✅ свою | ✅ | ✅ всех | ✅ |
| **Видеть дебиторку по клиенту** | ❌ | ✅ | ✅ | ✅ |
| **Настраивать справочники (ставки НДС)** | ❌ | ❌ | ❌ | ✅ |

## Принципы RBAC v1

1. **«Свои»** = где `Order.createdById = currentUser.id` (для менеджера).
2. **«Всех»** = бухгалтер/директор/админ видят все Order без фильтра по менеджеру.
3. **Закрытие двухступенчатое:** бухгалтер ставит «Готов к закрытию» → директор подтверждает (Q3 ✅ РЕШЕНО).
4. **Маржинальность для менеджера:** только своя (GAP-014 ✅ РЕШЕНО). Чужая — только director/accountant/admin.
5. **Дебиторка:** только для director/accountant/admin (менеджер НЕ видит).

## RBAC vs Бизнес-правила

- **RBAC** = «может ли роль выполнить действие» (например, может ли менеджер зарегистрировать платёж?).
- **Инвариант** = «при каких условиях действие валидно» (например, закрыть Order только если сальдо = 0). Это уже в [`05-biznes-pravila.md`](./05-biznes-pravila.md).
