## 0. Контекст

Финансы = финальный этап жизненного цикла сделки: фиксация оплаты + дебиторка + расчёт маржинальности + закрытие заказа. Главная сущность — `Order` (Order-XXXX), авто-создаётся при `Contract.status='SIGNED'` (СПОР-5 ✅). 5 сущностей: `Order` + `Invoice` (Inv-XXXX) + `Payment` (Pay-XXXX, INCOMING/STORNO) + `Refund` (Ref-XXXX, отдельная сущность по GAP-023 ✅) + `Counter`. **Storno vs Refund — это 2 разные сущности**, не путать: Storno = техническая корректировка опечатки бухгалтера, Refund = реальный возврат денег клиенту. 5 статусов Order: `DRAFT → IN_PROGRESS → AWAITING_PAYMENT → CLOSED → CANCELLED`.

5 папок модуля, `02-*` расщеплена на **2 воркспейс-папки** (по URL: `/finance/orders` + регистрация оплат) — Стратегия C.
