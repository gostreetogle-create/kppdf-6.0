# 05_Финансы/ — Модуль Финансы / Закрытие заказа

> **Назначение.** Документация модуля «Финансы / Закрытие заказа (OrderClosing)». Финальный этап цепочки: после производства и отгрузки — фиксация оплаты, сверка счётов, закрытие сделки.

## Структура модуля (после декомпозиции)

```
05_Финансы/
├── README.md                          # ← этот файл (entrypoint)
├── 00-spr/                            # Справочники (глоссарий, orgs, products, clients)
├── 05-konstruktor-finansov/           # Конструктор: счета, платежи, рефанды
├── 05-zhiznennyj-cikl/                # Жизненный цикл: статусы, налоговый учёт
└── 05-pravila/                        # Правила: RBAC + бизнес-правила
```

> **🎯 Цель для Run 5 Аналитика:** заполнить 3 STUB правилами (RBAC ≥30 + biznes-pravila ≥25 + statusy 5+5+2).

## Контракты с другими модулями

| Контракт | Trigger | Target |
|---|---|---|
| **Договор → Финансы** | `Contract.status='SIGNED'` (СПОР-5) | Авто-создание Order в статусе `DRAFT` → `IN_PROGRESS` |
| **Производство → Финансы** | `ProductionOrder.status='DONE'` | Все позиции выполнены → триггер для сверки |
| **Склад → Финансы** | `Shipment.status='SHIPPED'` | Себестоимость (costOfGoodsSold) для маржи |
| **Финансы → КП** | Закрытие Order | КП остаётся «Оплачено» (не возвращается) |
