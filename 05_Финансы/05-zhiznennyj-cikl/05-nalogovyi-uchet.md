> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §5](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение — Аналитик Run 5 (ТЗ-010).

## Назначение

Документирует **налоговый учёт** модуля Финансы: УСН (упрощённая система налогообложения), НДС (1 ставка 20% в v1), расчёт маржинальности. Источник: `МОДУЛЬ-ФИНАНСЫ.md` §6 «Маржа заказа» + §10 «Поля Invoice».

## НДС в v1

| Параметр | Значение | Источник |
|---|---|---|
| **Ставка** | 20% (одна на всё КП) | СПОР-15 ✅ РЕШЕНО |
| **Разделение** | Авто по `vatRate` из Договора | Q2 ✅ РЕШЕНО |
| **Multi-stavka** | Отложена в v2 | СПОР-15 |

**Формула НДС в Invoice:**
```
Invoice.amount (с НДС) = Invoice.amount (без НДС) × (1 + vatRate / 100)
```

## УСН (упрощённая система налогообложения)

| Параметр | Описание |
|---|---|
| **Объект налогообложения** | «Доходы минус расходы» (для маржинальности) |
| **Расходы** | Себестоимость из Склада (`costOfGoodsSold`) |
| **Доходы** | Сумма оплаченных Invoice (`paidAmount`) |
| **Ставка** | 15% (настраивается администратором) |

## Маржинальность (формула)

```
Маржа Order = Σ (цена продажи позиции × кол-во) − Σ costOfGoodsSold (из Склада)
Маржа % = (Маржа / Σ цена продажи) × 100
```

**Важные правила:**
- Себестоимость доступна только для **фактически отгруженных** позиций (`Shipment.status='SHIPPED'`).
- Услуги/работы (Product.kind = `SERVICE` / `WORK`) — отдельная строка, **без себестоимости**.
- Если позиция ещё не отгружена → маржа = `null` (не додумывать).
- Составной товар — итоговая себестоимость из Склада (уже агрегированная).

## Принципы наполнения (Аналитик Run 5)

1. **Минимум налоговой логики.** В v1 — только отображение НДС + базовый расчёт маржи. Полноценный налоговый учёт — v2.
2. **НДС одна ставка.** Multi-stavka (разные ставки для разных товаров) отложена в v2 (СПОР-15).
3. **Cross-ref на Склад:** себестоимость — читатель, не создатель. Финансы НЕ модифицируют `costOfGoodsSold`.
