# 05_Финансы/05-zhiznennyj-cikl/05-statusy.md — Статусы Order, Invoice, Payment

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §5](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение — Аналитик Run 5 (ТЗ-010).

## Назначение

Документирует **все статусы** финансовых сущностей: Order (5 статусов), Invoice (5 статусов), Payment (2 типа). Каждый статус имеет точное определение, когда наступает, что можно делать дальше, цветовую метку UI. Источник: `МОДУЛЬ-ФИНАНСЫ.md` §3 «Жизненный цикл».

## Order — 5 статусов

### 1. `DRAFT` (Черновик)

| Поле | Значение |
|---|---|
| Когда | Авто-создан при `Contract.status='SIGNED'` (СПОР-5) |
| Что можно | Редактировать (пока не переведён в IN_PROGRESS автоматически) |
| Цвет UI | серый |
| RBAC | admin, director |

### 2. `IN_PROGRESS` (В работе)

| Поле | Значение |
|---|---|
| Когда | Автоматически через 1 секунду после DRAFT (идёт производство / ожидаем отгрузку) |
| Что можно | Выставлять счета (Invoice), регистрировать платежи |
| Цвет UI | синий |
| RBAC | admin, accountant, director |

### 3. `AWAITING_PAYMENT` (Ожидает оплаты)

| Поле | Значение |
|---|---|
| Когда | Все позиции отгружены (Shipment.status='SHIPPED') + выставлен финальный Invoice |
| Что можно | Регистрировать платежи, закрывать заказ (если сальдо = 0) |
| Цвет UI | жёлтый |
| RBAC | admin, accountant, director |

### 4. `CLOSED` (Закрыт)

| Поле | Значение |
|---|---|
| Когда | Все счета оплачены (Invoice.balance = 0 для всех Invoice) + задержка 1 день для корректировок |
| Что можно | Ничего (архив) |
| Цвет UI | зелёный |
| RBAC | admin, director (закрытие двухступенчатое — Q3 ✅ РЕШЕНО) |

### 5. `CANCELLED` (Отменён) — добавлено 24.06.2026

| Поле | Значение |
|---|---|
| Когда | ЗК отменён + Refund на всю сумму (правка #3 МАСТЕР-АУДИТ-V6.md) |
| Что можно | Ничего (финальный статус, аудит) |
| Цвет UI | красный |
| RBAC | admin, director |

**Таблица переходов Order:**

| Статус | Следующий | Триггер | Кто может перевести |
|---|---|---|---|
| DRAFT | IN_PROGRESS | Автоматически (через 1 сек) | Автоматически |
| IN_PROGRESS | AWAITING_PAYMENT | Все позиции shipped + Invoice выставлен | Автоматически |
| AWAITING_PAYMENT | CLOSED | Сальдо = 0 (все Invoice fully paid) | Бухгалтер / автоматически |
| (Любой после DRAFT) | CANCELLED | ЗК отменён + Refund на всю сумму | Бухгалтер (через Refund) |

## Invoice — 5 статусов

### 1. `DRAFT` (Черновик)

| Поле | Значение |
|---|---|
| Когда | Счёт создан, но не выставлен |
| Что можно | Редактировать, выставить, удалить |
| Цвет UI | серый |

### 2. `ISSUED` (Выставлен)

| Поле | Значение |
|---|---|
| Когда | Счёт отправлен клиенту |
| Что можно | Регистрировать платежи |
| Цвет UI | синий |

### 3. `PARTIALLY_PAID` (Частично оплачен)

| Поле | Значение |
|---|---|
| Когда | Поступила часть суммы (Payment < amount) |
| Что можно | Регистрировать дополнительные платежи |
| Цвет UI | жёлтый |

### 4. `FULLY_PAID` (Полностью оплачен)

| Поле | Значение |
|---|---|
| Когда | Сальдо = 0 (paidAmount = amount) |
| Что можно | Ничего (финальный) |
| Цвет UI | зелёный |

### 5. `OVERDUE` (Просрочен)

| Поле | Значение |
|---|---|
| Когда | dueDate + 1 день при сальдо > 0 |
| Что можно | Регистрировать платежи (но выделяется красноватым) |
| Цвет UI | красный |

---

## Payment — 2 типа

| Тип | Описание | `amount` | Влияние на сальдо |
|---|---|---|---|
| `INCOMING` | Входящий платёж от клиента | `> 0` | `Invoice.paidAmount += amount` |
| `STORNO` | Исправление ошибочного платежа | `< 0` | `Invoice.paidAmount += amount` ( amount < 0 → уменьшает) |

## Принципы наполнения (Аналитик Run 5)

1. **Один status = одно определение.** Не синонимы. Если `CLOSED` означает разное в КП и Финансах — явно прокомментировать.
2. **CANCELLED = финальный.** Никакой переход из CANCELLED в живой статус.
3. **Цветовая схема** — должна совпадать с КП/Договор/Производство/Склад (серый → синий → жёлтый → зелёный → красный).
