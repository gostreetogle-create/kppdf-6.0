# 05_Финансы/05-konstruktor-finansov/05-platezhi.md — Конструктор платежей (Payment)

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §5](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение — Аналитик Run 5 (ТЗ-010).

## Назначение

Документирует **конструктор платежей (Payment)** — как бухгалтер регистрирует входящие платежи и сторнирует ошибки. Источник: `МОДУЛЬ-ФИНАНСЫ.md` §4 (шаг 3) + §6 «Правила Сторно» + §10 «Поля Payment».

## 2 типа Payment

| Тип | Описание | `amount` | Когда |
|---|---|---|---|
| `INCOMING` | Входящий платёж от клиента | `> 0` | Деньги поступили |
| `STORNO` | Исправление ошибочного платежа | `< 0` | Опечатка бухгалтера |

## Поля Payment (from §10)

| Поле | Тип | Зачем |
|---|---|---|
| `paymentId` | `Pay-XXXX` | Уникальный номер (СПОР-13: отдельный счётчик) |
| `type` | enum (INCOMING \| STORNO) | Default = INCOMING |
| `orderId` | FK → Order | К какому заказу (ON DELETE RESTRICT) |
| `invoiceId` | FK → Invoice (опц.) | К какому счёту (СПОР-7: ручной выбор) |
| `correctsPaymentId` | FK → Payment (опц.) | Только для STORNO: ссылка на ошибочный платёж |
| `receivedAt` | date | Когда деньги фактически пришли (Q7 ✅ РЕШЕНО) |
| `registeredAt` | timestamp | Когда бухгалтер внёс в систему |
| `amount` | decimal | INCOMING ⇒ > 0. STORNO ⇒ < 0 |
| `method` | enum | Безнал / нал / карта (ручной ввод v1) |
| `notes` | text | NOT NULL при type=STORNO |
| `createdById` | FK → User | Кто внёс |

## Правила Сторно (from §6)

1. **Нулевое влияние на бизнес-статусы:** ЗК/Договор/Order НЕ меняются.
2. **Сальдо возвращается в исходное:** `Invoice.paidAmount` и `Order.balance` восстанавливаются.
3. **Запрет на цепочки:** Нельзя сделать STORNO на STORNO.
4. **Обязателен комментарий:** `Payment.notes NOT NULL` при type='STORNO'.
5. **В UI** отображается рядом с ошибочным платежом серой/перечёркнутой строкой.

## Сальдо (формула)

```
Invoice.balance = Invoice.amount − Σ Payment.amount(type=INCOMING) + Σ abs(Payment.amount(type=STORNO))
```

## Принципы наполнения (Аналитик Run 5)

1. **Только бухгалтер/директор** может регистрировать платежи (RBAC §6).
2. **SPOR-7:** ручной выбор Invoice через dropdown. Без авто-FIFO.
3. **SPOR-13:** отдельный счётчик `Counter.type='payment'`.
