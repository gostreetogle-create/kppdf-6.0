# 05_Финансы/05-konstruktor-finansov/05-scheta.md — Конструктор счетов (Invoice)

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §5](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение — Аналитик Run 5 (ТЗ-010).

## Назначение

Документирует **конструктор счетов (Invoice)** — как бухгалтер/директор создаёт и управляет счетами клиенту. Источник: `МОДУЛЬ-ФИНАНСЫ.md` §4 (шаг 2) + §10 «Поля Invoice».

## 3 типа Invoice

| Тип | Описание | Когда создаётся |
|---|---|---|
| `ADVANCE` (Аванс) | Счёт на предоплату (например, 50% от суммы договора) | До начала производства |
| `MAIN` (Основной) | Финальный счёт на всю сумму или остаток | После отгрузки |
| `ADJUSTMENT` (Корректировочный) | Корректировка суммы (изменение цен, доп.работы) | По необходимости |

## 5 статусов Invoice

| Статус | Описание | Когда |
|---|---|---|
| `DRAFT` (Черновик) | Счёт создан, но не выставлен | Бухгалтер готовит счёт |
| `ISSUED` (Выставлен) | Счёт отправлен клиенту | После «Отправить счёт» |
| `PARTIALLY_PAID` (Частично оплачен) | Поступила часть суммы | Payment < amount |
| `FULLY_PAID` (Полностью оплачен) | Сальдо = 0 | Все платежи поступили |
| `OVERDUE` (Просрочен) | Срок оплаты прошёл, сальдо > 0 | dueDate + 1 день |

## Поля Invoice (from §10)

| Поле | Тип | Зачем |
|---|---|---|
| `invoiceId` | `Inv-XXXX` | Уникальный номер (СПОР-13: отдельный счётчик) |
| `orderId` | FK → Order | К какому заказу (ON DELETE CASCADE) |
| `issuedAt` | date | Когда выставлен |
| `dueDate` | date | Крайний срок оплаты |
| `amount` | decimal | Сумма счёта |
| `paidAmount` | decimal | Сколько поступило |
| `balance` | decimal | `amount - paidAmount` |
| `status` | enum | Статус (5 вариантов выше) |
| `invoiceType` | enum | Тип (ADVANCE / MAIN / ADJUSTMENT) |

## Экраны UI (from §5)

- **Список счетов** — табличный вид в карточке Order (№, дата, сумма, статус, остаток).
- **Создание счёта** — форма: выбор типа (аванс/основной/корректировочный), сумма, срок оплаты.
- **Карточка счёта** — реквизиты, история платежей, сальдо.

## Принципы наполнения (Аналитик Run 5)

1. **Только бухгалтер/директор** может создавать/редактировать счёт (RBAC §6).
2. **НДС:** авто разделение по `vatRate` Договора (Q2 ✅ РЕШЕНО).
3. **Нумерация:** отдельный счётчик `Counter.type='invoice'` (СПОР-13).
