# 05_Финансы/00-spr/00-glossary.md — Глоссарий терминов модуля «Финансы»

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §5](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение содержанием — Аналитик Run 5 (ТЗ-010).

## Назначение

Глоссарий терминов модуля `Финансы` — единый источник для всех downstream-документов. Все термины должны совпадать с [`../../99_Справочники/GLOSSARY-MASTER.md`](../../99_Справочники/GLOSSARY-MASTER.md).

## 7 ключевых терминов (из `МОДУЛЬ-ФИНАНСЫ.md` §0)

| Термин | Что значит |
|---|---|
| **Order (Заказ)** | Сквозная сущность «от КП до оплаты». Уникальный номер **Order-XXXX**. Авто-создаётся при `Contract.status='SIGNED'` (СПОР-5). |
| **Invoice (Счёт)** | Документ выставления оплаты клиенту. Типы: аванс / основной / корректировочный. Уникальный номер **Inv-XXXX** (СПОР-13). |
| **Payment (Платёж)** | Факт поступления денег от клиента. Типы: `INCOMING` (входящий) / `STORNO` (сторно — исправление опечатки). Уникальный номер **Pay-XXXX** (СПОР-13). |
| **Refund (Возврат)** | Отдельная сущность возврата денег клиенту (НЕ отрицательный Payment — GAP-023 ✅ РЕШЕНО). Номер **Ref-XXXX**. |
| **OrderClosing (Закрытие заказа)** | Процедура финальной сверки: все позиции отгружены, все счета оплачены, сальдо = 0. |
| **Balance (Сальдо)** | Разница выставленного (Invoice) и полученного (Payment). Отрицательное = клиент должен. Положительное = переплата. |
| **Себестоимость (costOfGoodsSold)** | Суммарная себестоимость позиций из модуля Склад (`Shipment.costOfGoodsSold`). Для расчёта маржинальности. |

## Дополнительные термины

| Термин | Что значит |
|---|---|
| **Маржа (Margin)** | `Σ (цена продажи × кол-во) − Σ costOfGoodsSold`. Услуги/работы — отдельная строка без себестоимости. |
| **Дебиторка** | Сумма отрицательных `Order.balance` по всем активным (`IN_PROGRESS` + `AWAITING_PAYMENT`) Order. |
| **Сторно (STORNO)** | `Payment.type='STORNO'` — техническая корректировка опечатки бухгалтера. `amount < 0`, `correctsPaymentId` FK. Деньги физически не двигались. |

## Типы статусов (cross-ref)

Полный список статусов — в [`../05-zhiznennyj-cikl/05-statusy.md`](../05-zhiznennyj-cikl/05-statusy.md). Краткая сводка:

**Order (5 статусов):**
- `DRAFT` (Черновик)
- `IN_PROGRESS` (В работе)
- `AWAITING_PAYMENT` (Ожидает оплаты)
- `CLOSED` (Закрыт)
- `CANCELLED` (Отменён — добавлено 24.06.2026)

**Invoice (5 статусов):**
- `DRAFT` (Черновик)
- `ISSUED` (Выставлен)
- `PARTIALLY_PAID` (Частично оплачен)
- `FULLY_PAID` (Полностью оплачен)
- `OVERDUE` (Просрочен)

**Payment (2 типа):**
- `INCOMING` (Входящий платёж)
- `STORNO` (Сторно — исправление)

## Принципы наполнения (Аналитик Run 5)

1. **Один термин — одно определение.** Если термин пересекается с Складом/Производством — дать cross-ref, не дублировать.
2. **Refund ≠ Payment.** Явно подчеркнуть: Refund — отдельная сущность (GAP-023 ✅ РЕШЕНО), НЕ отрицательный Payment.
3. **RUB жёстко.** Ссылка на СПОР-14: `currency = 'RUB'` в v1, мультивалюта отложена в v2.
