# 05_Финансы/00-spr/00-products.md — Продукты / себестоимость модуля «Финансы»

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §5](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение — Аналитик Run 5 (ТЗ-010).

## Назначение

Документирует **связь Финансов с себестоимостью товаров** из модуля Склад. Финансы НЕ создают продуктов — они только читают `costOfGoodsSold` для расчёта маржинальности. Источник: `МОДУЛЬ-ФИНАНСЫ.md` §6 «Маржа заказа».

## Связь с Складом

| Источник | Поле | Использование в Финансах |
|---|---|---|
| `Shipment.costOfGoodsSold` | decimal | Себестоимость отгруженных позиций для маржи |
| `ProductionOrder.status='DONE'` | enum | Все ЗП выполнены → позиции доступны для себестоимости |
| `Shipment.status='SHIPPED'` | enum | Позиция отгружена → себестоимость фиксируется |

## Маржинальность (формула)

```
Маржа Order = Σ (цена продажи позиции × кол-во) − Σ costOfGoodsSold
```

**Важные правила:**
- Себестоимость доступна только для **фактически отгруженных** позиций (`Shipment.status='SHIPPED'`).
- Услуги/работы (Product.kind = `SERVICE` / `WORK`) — отдельная строка в марже, **без себестоимости**.
- Составной товар (5 модулей) — берётся **итоговая себестоимость** из Склада (уже агрегированная).

## Принципы наполнения (Аналитик Run 5)

1. **Финансы — читатель, не создатель.** Никакого CRUD на продукты. Только чтение FK из Shipment.
2. **null = «не рассчитана».** Если позиция ещё не отгружена — маржа = `null`, не додумывать.
3. **Mirror Склад:** cross-ref на `04_Склад/00-spr/00-products.md` для полного описания себестоимости.
