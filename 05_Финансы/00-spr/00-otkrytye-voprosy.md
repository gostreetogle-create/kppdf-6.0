| **OQ-001** | Точные триггеры для `Refund` (отмена ЗК в Производстве? расторжение Договора? возврат товара клиентом?). Какие источники и каскады? | cross-module | → `CROSS-MODULE-OQ.md` §1 (Refund) | ⚠️ базовое |

## 2. Направление ответов для Аналитика

| OQ | Рекомендуемая гипотеза (не финальная — обоснование в Run1 Аналитика) |
|---|---|
| OQ-001 | 3 триггера: (1) `ProductionOrder.status='CANCELLED'` после `paid`; (2) `Contract.status='TERMINATED'` после частичной оплаты; (3) возврат товара через `WriteOffAct.reason='DEFECT'`. Каждый → отдельный `Refund` с явной ссылкой на источник. |
| OQ-002 | Бухгалтер оформляет `Refund` (СПОР-12 + GAP-023 ✅). Менеджер только сигнализирует. |
| OQ-003 | Один `Payment` → один `Invoice` в базовом сценарии (СПОР-7 ✅). Для нераспределённого платежа — `InvoiceId IS NULL` (привязка к Order в целом). FIFO-распределение отложено в v2. |
| OQ-004 | Закрытый Order может быть ОТКРЫТ повторно при позднем Shipment (через `Order.status='CLOSED' → 'AWAITING_PAYMENT'` rollback на 1 день). После — пересчёт `margin` разморожен. |
| OQ-005 | Авто при `balance=0` через trigger (СПОР-7 ✅) — если в UI всё же есть ручная кнопка, она дублирует авто (для бухгалтерского контроля). |
