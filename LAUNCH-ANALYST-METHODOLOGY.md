# LAUNCH-ANALYST-METHODOLOGY.md — Package для запуска Бизнес-аналитика (Pipeline v6 Run 0/5: методология)

> **Назначение.** Готовый copy-paste пакет для запуска Бизнес-аналитика в **новой сессии Codebuff** для применения **3 методологических улучшений**, отложенных в [`PROJECT-STATE-LOG.md` PSL-007](PROJECT-STATE-LOG.md). **Это специальный Run 0/5** — scope не привязан к одному модулю, а universal: scoring постфактум применяется к 15 СПОР + 38 Q, шаблон СПОРА идёт в начало `СПОРНЫЕ-МОМЕНТЫ.md`, новый файл `99_Справочники/DECISION-METHODOLOGY.md` создаётся как портируемый алгоритм.
>
> **Когда использовать.** Когда PO хочет применить scoring к закрытым решениям. **Не запускать параллельно с Run 1/5 КП** (другая роль, другой scale: методология ≠ наполнение правил КП).
>
> **Hard-limit Override.** Этот launch-package сознательно сделан короче `LAUNCH-ANALYST.md` (~200 строк vs ~250) — он проще, поскольку scope = универсальный, не зависит от модуля.

1. **Открыть новую сессию Codebuff** (`codebuff` → New Chat).

   | Критерий | 1 | 5 |
   |---|---|---|
   | **Бизнес-ценность** | минимальная | критическая для MVP |
   | **Сложность** | тривиальная | значительные ресурсы |
   | **Риски** | безопасно | высокий (финансовый / юридический) |
   | **Совместимость** | конфликтует с архитектурой | полностью соответствует |
   | **Время** | мгновенно | дни/недели |

   Интерпретация итогового балла: **>20** = сильное решение, MVP-ready;
   **15-20** = хорошее решение; **10-14** = среднее; **<10** = спорное или v2.

4. **ШАБЛОН нового СПОРА** (в начало `99_Справочники/СПОРНЫЕ-МОМЕНТЫ.md` §0

   | **ID** | `СПОР-NEW` |
   | **Дата** | YYYY-MM-DD |
   | **Роль** (кто инициировал) | Архитектор / Бизнес-аналитик / Моделировщик / UX / QA / Координатор / PO |
   | **Суть** | 1-2 предложения — о чём спор |
   | **Контекст** | почему спор возник (связь с OQ / документом) |
   | **Варианты** | A / B / C / свой (свободный) |
   | **Кто решает** | PO / Консилиум ролей |
   | **Тайм-бокс** | целевая дата принятия решения |
   | **Статус** | 🔵 OPEN / 🟡 PENDING DECISION / 🟢 RESOLVED |

   Шаблон должен быть **готов к копированию в новые СПОРЫ** (один
   пример заполненного СПОРА-NEW для демонстрации формата).

5. **`DECISION-METHODOLOGY.md`** — новый портируемый документ
   (`99_Справочники/DECISION-METHODOLOGY.md`, ~250 строк target). 5 фаз
   (явно из PSL-007):

   1. **Контейнер** — что за проблема + границы scope
