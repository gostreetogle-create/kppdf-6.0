# 02_Договор/03-zhiznennyj-cikl/03-statusy.md — 7 статусов Договора

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §2](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение — Аналитик Run 2 (ТЗ-007).

## Назначение

Документирует **7 статусов Договора** (6 baseline + TERMINATED с 24.06.2026). Каждый статус имеет точное определение, когда наступает, что можно делать дальше, цветовую метку UI.

## 7 статусов

### 1. `DRAFT` (Черновик)

| Поле | Значение |
|---|---|
| Когда | Только что сконвертирован из КП |
| Что можно | Редактировать, отправить клиенту, архивировать |
| Цвет UI | серый |
| RBAC | admin, manager (свой), director |

### 2. `SENT` (Отправлен клиенту)

| Поле | Значение |
|---|---|
| Когда | Договор ушёл клиенту на подпись |
| Что можно | Получить подпись, отозвать/скорректировать |
| Цвет UI | синий |
| RBAC | admin, manager, director |

### 3. `SIGNED_CLIENT` (Подписан клиентом)

| Поле | Значение |
|---|---|
| Когда | Клиент подписал, Договор действует |
| Что можно | Связать с Производственным заказом, вернуть в работу (ошибка) |
| Цвет UI | зелёный |
| RBAC | admin, manager, director |

### 4. `IN_PROGRESS` (В работе)

| Поле | Значение |
|---|---|
| Когда | Договор действует, по нему ведутся работы (Производственный заказ в активных статусах) |
| Что можно | Следить за ЗК, отметить завершение |
| Цвет UI | жёлтый |
| RBAC | все роли (= read-only обычно) |

### 5. `COMPLETED` (Завершён)

| Поле | Значение |
|---|---|
| Когда | Обязательства выполнены: товар отгружен, услуги оказаны, оплата пришла |
| Что можно | Архивировать (авто через 90 дней или ручной) |
| Цвет UI | тёмно-зелёный |
| RBAC | admin |

### 6. `ARCHIVED` (Архив)

| Поле | Значение |
|---|---|
| Когда | Мягкое удаление (не показывается в основном списке) |
| Что можно | Восстановить (если ошибка) |
| Цвет UI | серый |
| RBAC | admin |

### 7. `TERMINATED` (Расторгнут) — добавлен 24.06.2026

| Поле | Значение |
|---|---|
| Когда | Договор аннулирован до полного исполнения |
| Что можно | Ничего (финальный статус) |
| Цвет UI | красный |
| RBAC | admin, director |

**Эффект:** Order в Финансах переходит в `CANCELLED` (см. `МОДУЛЬ-ДОГОВОР.md` §11 → `МОДУЛЬ-ФИНАНСЫ.md` §6 «Правила Refund»).

## Сравнение с КП-статусами

| КП | Договор |
|---|---|
| DRAFT | DRAFT |
| — | SENT (специфично для Договора — подписание клиентом) |
| SENT_TO_CLIENT | — |
| ACCEPTED | — |
| CONVERTED | SIGNED_CLIENT (после подписания) |
| PAID | IN_PROGRESS |
| COMPLETED | COMPLETED |
| ARCHIVED | ARCHIVED |
| CANCELLED | TERMINATED |

## Принципы наполнения (Аналитик Run 2)

1. **Один status = одно определение.** Не синонимы. Если `COMPLETED` означает разное в КП и Договоре — явно прокомментировать.
2. **TERMINATED = финальный.** Никакой переход из TERMINATED в живой статус.
3. **Цветовая схема** — должна совпадать с КП (если COMPLETED = тёмно-зелёный для обоих).
