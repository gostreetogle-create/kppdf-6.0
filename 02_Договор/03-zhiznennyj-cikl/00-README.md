## Назначение

Папка **жизненного цикла Договора** — статусы, переходы, пролонгация, неустойки. Это **state machine**, которая управляет всем поведением Договора во времени.

## State machine (краткая схема)

```
         ┌─────────────┐
         │  DRAFT      │ ← конвертирован из КП
         └──────┬──────┘
                │ Отправить клиенту
                ▼
         ┌─────────────┐
         │  SENT       │ (отправлен клиенту)
         └──────┬──────┘
                │ Клиент подписал
                ▼
         ┌─────────────┐
         │ SIGNED_CLIENT │
         └──────┬──────┘
                │ Договор в работе
                ▼
         ┌─────────────┐
         │ IN_PROGRESS │ ← связан с активным
         └──────┬──────┘    Производственным заказом
                │ Производство завершено
                ▼
         ┌─────────────┐
         │ COMPLETED   │
         └──────┬──────┘
                │ Архивировать (авто или ручной)
                ▼
            ARCHIVED

  В любом статусе (кроме ARCHIVED): можно → TERMINATED
  TERMINATED ←── Order.cancel + Refund в Модуле Финансы
```

## Принципы наполнения (Аналитик Run 2)

1. **State machine = ядро.** Бизнес-правила в [`../04-pravila/04-biznes-pravila.md`](../04-pravila/04-biznes-pravila.md) ссылаются на конкретные переходы здесь.
2. **Только разрешённые переходы.** Невалидный переход = ошибка UI + audit log.
3. **TERMINATED** = финальный с 24.06.2026 (СПОР-3 в МОДУЛЬ-доке). Никаких возвратов из TERMINATED.
