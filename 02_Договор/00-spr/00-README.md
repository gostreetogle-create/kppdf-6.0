## Принципы наполнения (Аналитик Run 2)

1. **Snapshot semantics:** все поля юр-лиц, ФИО подписантов, реквизиты — фиксируются в момент подписания. См. `МОДУЛЬ-ДОГОВОР.md` §5.5.
2. **Hard-link на КП:** Клиент Договора = Клиент КП-источника (через `Contract.parentProposalId → Proposal.id`). Не дублировать данные из `01_КП/00-spr/`.
3. **5 baseline OQ** уже стоят в `00-otkrytye-voprosy.md` — если новые вопросы возникают → добавлять в `99_Справочники/OPEN-QUESTIONS-MASTER.md`, не сюда.
4. **Контрактные специфики** (в `00-orgs.md`):
   - Наша организация-продавец (`Contract.contractorId`)
   - Клиент-покупатель (`Contract.customerId`)

5. **Модульная система галочек** RBAC — наследуется от `01_КП/04-pravila/04-rbac.md` с расширениями.
