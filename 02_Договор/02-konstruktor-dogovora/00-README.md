## Назначение

Папка **сборки Договора** — содержит бизнес-логику экранов конструктора, кнопок действий, формы редактирования реквизитов, выбора типа Договора. Это то, что **видит и использует менеджер** при работе с Договором.

## Главные экраны конструктора

### 1. Форма создания Договора (при конвертации КП)

- **Выбор типа Договора** (radio: со Спецификацией / Рамочный)
- **Выбор шаблона** (dropdown из `01-shablon/`)
- **Preview** — показывает первые строки Договора с подставленными реквизитами
- **Кнопка «Создать»**

### 2. Редактор Договора (`/contracts/[id]`)

3-зонный макет (зеркалит [`../../01_КП/02-konstruktor-kp/`](../../01_КП/02-konstruktor-kp/)):

- **В центре**: живой preview А4-документа.
- **Слева**: dropdown разделов пакета (Товары из КП-источника, Шаблоны документов).
- **Справа**: группа «Основное» + группа «Параметры» + группа «Кнопки действий» (Сохранить, Отправить клиенту, Архивировать, Создать новую версию, Добавить Спецификацию).

## Принципы наполнения (Аналитик Run 2)

1. **Mirror КП:** стиль, layout, RBAC-panels — copy-paste от [`../../01_КП/02-konstruktor-kp/`](../../01_КП/02-konstruktor-kp/) с подстановкой Договор-конкретных полей.
2. **Одна кнопка = одно действие.** Не смешивать «Сохранить и отправить» — это плохой UX.
3. **Live preview** — обязателен. Менеджер не должен угадывать «как это будет выглядеть в PDF».
