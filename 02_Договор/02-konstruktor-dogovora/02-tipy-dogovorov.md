# 02_Договор/02-konstruktor-dogovora/02-tipy-dogovorov.md — Типы Договоров

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECOMPOSITION-PLAN.md §2](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение — Аналитик Run 2 (ТЗ-007).

## Назначение

Документирует **типы Договоров**, поддерживаемые в v1: «Договор поставки со Спецификацией» (default), «Рамочный», «Договор с подрядом». Источник: `МОДУЛЬ-ДОГОВОР.md` §2.

## 3 типа Договоров v1

### 1. Договор поставки со Спецификацией (default)

**Что:** Стандартный Договор с зафиксированной Спецификацией (товары + цены + количество).

| Поле | Поведение |
|---|---|
| `ContractItem.priceWithoutVat` | обязателен |
| `ContractItem.quantity` | обязателен |
| Срок действия | явный `validUntil` |
| Применяется | 90% реальных сделок |

**Когда выбирать:** клиент хочет видеть конкретные цены и количества в подписанном документе. Юридически «полная фиксация условий».

### 2. Рамочный договор

**Что:** Договор фиксирует **номенклатуру и общее направление**, но цены согласовываются по факту поставки.

| Поле | Поведение |
|---|---|
| `ContractItem.priceWithoutVat` | **NULL** (цен нет в Спецификации) |
| `ContractItem.quantity` | обязателен (общее количество) |
| Срок действия | обычно 1 год с пролонгацией |
| Применяется | долгосрочные поставки (стройматериалы, расходники) |

**Когда выбирать:** клиент хочет долгосрочное сотрудничество, но конкретные цены меняются от поставки к поставке. Юридически «обязательство поставить, цена — дополнительно».

**ВАЖНО:** в рамочном Договоре Спецификация НЕ содержит цен, но `packageTag` всё равно присваивается для фильтра-представления «Картотека сделки».

### 3. Договор с подрядом (subcontracting)

**Что:** Наша организация — генеральный подрядчик. Часть работ передаём субподрядчикам.

| Поле | Поведение |
|---|---|
| `ContractItem.priceWithoutVat` | обязателен |
| `ContractItem.quantity` | обязателен |
| Доп. поля `Contract.subcontractorIds[]` | FK → Organization[] |

**v1 ограничение:** подрядчики фиксируются **только в Org-справочнике** (не в Спецификации). Детальное распределение работ между подрядчиками — v2.

## Правила выбора типа

1. **Тип выбирается при создании** (мини-форма при конвертации КП → Договор).
2. **Изменить тип** — нельзя. Если ошибся → «Создать новую версию» с правильным типом.
3. **Один тип на всю жизнь Договора.** Нельзя «был рамочный → стал со Спецификацией».
