# 02_Договор/02-konstruktor-dogovora/02-rekvizity.md — Реквизиты сторон

> ⚠️ **STUB.** Создан декомпозицией (см. [MODULE-DECОMPOSITION-PLAN.md §2](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)). Наполнение — Аналитик Run 2 (ТЗ-007).

## Назначение

Документирует **реквизиты сторон** Договора (продавец + клиент), их snapshot-логику и токены подстановки в шаблон. Источник: `МОДУЛЬ-ДОГОВОР.md` §5.5 + §7.

## Snapshot-логика реквизитов

> **Критическое правило:** все реквизиты фиксируются **в момент подписания**. Изменение в Org после подписания **НЕ влияет** на подписанный Договор.

**Почему:** юридически подписанный документ нельзя «исправить задним числом» через изменение справочника. Это защищает от ситуации «директор сменился → старый Договор автоматически показывает нового директора» (юридически опасно).

## Поля реквизитов

### Продавец (наша организация) — `Contract.contractorId`

12 полей snapshot (см. [`../00-spr/00-orgs.md`](../00-spr/00-orgs.md) таблица 1):

- `name`, `nameFull`, `inn`, `kpp`, `ogrn`
- `legalAddress`, `actualAddress`
- `bankName`, `bankBik`, `bankAccount`
- `directorName`, `directorPosition`

### Покупатель (клиент) — `Contract.customerId`

Те же 12 полей.

### Особые случаи

| Случай | Поведение |
|---|---|
| ИП клиент вместо ООО | `kpp = NULL` (у ИП нет КПП) |
| Договор с физлицом | `ogrn = NULL`, `kpp = NULL`, директор = само физлицо |
| Иностранный клиент | Currency ≠ RUB (но это **отложено** в v2, см. [00-out-of-scope.md](../00-spr/00-out-of-scope.md) #7) |

## Правила для реквизитов (v1)

1. **ФИО подписанта** — snapshot в момент подписания (`signedByClientAt` / `signedByContractorAt`). Директор сменился → в подписанных Договорах остаётся старый.
2. **Печать и логотип** — НЕ snapshot. Берутся из `Organization` в момент формирования PDF (это картинка, можно обновить).
3. **Email / телефон контактного лица** — snapshot (для уведомлений, актуально).
