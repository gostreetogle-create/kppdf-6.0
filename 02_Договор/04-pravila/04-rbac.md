## Назначение

Документирует **RBAC матрицу** модуля «Договор»: какая роль может выполнять какое действие. Источник: `МОДУЛЬ-ДОГОВОР.md` §8 RBAC-таблица.

| **Видеть список Договоров** | ✅ все | ✅ свои | ✅ все | ❌ | ❌ | ✅ только в цепочке | ✅ все (read-only) |

## Принципы для RBAC v1

1. **«Свои»** = где `createdById = currentUser.id`.
2. **«Только в цепочке»** для accountant = Договоры, к которым привязан `Order` через `Order.contractId`.
3. **Терминировать** = повышенные права (только admin / director). Менеджер НЕ может расторгнуть.
4. **Модульная система галочек** — админ отмечает, какие действия доступны для каждой роли в конкретной организации (для multi-tenancy).

## Правила для Терминирования (повышенные)

1. **Только admin / director** (manager НЕ может).
2. **Письменное уведомление клиента** — обязательно (сохранить файл-доказательство в `DocumentPackage`).
3. **Side-effect:** `Order.status = CANCELLED` в Финансах (см. `МОДУЛЬ-ДОГОВОР.md` §11).
4. **Без возврата** — TERMINATED = final, никаких восстановлений.

## RBAC vs Бизнес-правила

- **RBAC** = «может ли роль выполнить действие» (например, может ли менеджер Подписать?).
- **Инвариант** = «при каких условиях действие валидно» (например, `DRAFT → SENT` только если есть хотя бы 1 Item). Это уже в [`04-biznes-pravila.md`](./04-biznes-pravila.md).
