## Назначение

Полная карта 8 статусов ЗК + state machine переходов. Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §2.

## Статусы (8)

| Статус | Что значит | Кто двигает | Условие перехода |
|---|---|---|---|
| `CREATED` (Создан) | Только что создан из КП «Оплачено» | авто-система | автоматически |
| `PLANNED` (Планируется) | Начальник распределяет по цехам | начальник производства | вручную |
| `WAITING_MATERIALS` (Ожидает материалов) | Материалов/комплектующих не хватает | кладовщик/начальник | если StockRecord < required |
| `IN_PROGRESS` (В работе) | Задачи распределены, мастера выполняют | мастер | вручную |
| `PARTIAL` (Часть готова) | ≥50% задач выполнены, но не все | мастер | автоматически по quantityActual |
| `COMPLETED` (Завершён) | Все задачи выполнены | начальник | автоматически khi все quantityActual = quantityPlanned |
| `CANCELLED` (Отменён) | Заказ отменён | начальник/директор | вручную |
| `ARCHIVED` (Архив) | Давно завершён/отменён | авто/админ | через 90 дней или вручную |

## План наполнения (для Аналитика Run 3)

- Детальные условия переходов (какие именно проверки).
- Связь «Завершён» → автоматический StockMovement: Приход.
- «Отменён» → сигнал в Финансы (Refund, СПОР-12).
- «Архив» → `isActive = false` (единообразно с КП/Договором).
- Антипаттерн: дублирование с `03-perehody.md` → здесь только определения статусов.
