# 03_Производство/03-zhiznennyj-cikl/03-vozvrat-i-brak.md — Возврат и брак

> Контент будет наполнен Аналитиком Run 3 (ТЗ-008).

## Назначение

Обработка бракованных изделий и возвратов. Cross-ref на модуль Склад: `WriteOffAct` для списания брака.

## Контекст

Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §7 (возврат/брак) + §6 (связь со Складом): при обнаружении брака — задача не завершается или частично завершается с `quantityActual` < `quantityPlanned`. Списание брака → `WriteOffAct` в модуле Склад.

## План наполнения (для Аналитика Run 3)

- Механика брака: мастер фиксирует `quantityActual` < `quantityPlanned` → причина брака.
- Связь с QA (`03-kontrol-kachestva.md`): брак обнаружен при проверке.
- WriteOffAct: движение списания брака на Складе (cross-ref `МОДУЛЬ-СКЛАД-ПОДРОБНЫЙ.md` §6).
- Возврат от клиента: отдельный процесс (не через ProductionOrder).
- MVP boundary: нет автоматического пересчёта себестоимости при браке.
