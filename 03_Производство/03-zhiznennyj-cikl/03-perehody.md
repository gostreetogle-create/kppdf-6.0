# 03_Производство/03-zhiznennyj-cikl/03-perehody.md — Переходы между статусами

> ⚠️ **STUB.** Создан декомпозицией модуля `03_Производство/` (см. [MODULE-DECOMPOSITION-PLAN.md §3](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)).
> Контент будет наполнен Аналитиком Run 3 (ТЗ-008).

## Назначение

Разрешённые переходы между 8 статусами ЗК + RBAC-правила: кто может переводить из какого статуса в какой.

## Контекст

Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §2 (диаграмма переходов) + §5.3 (правила переходов): «Завершён» ставится автоматически при выполнении ВСЕХ задач. «Отменён» — ТОЛЬКО начальник/директор. «Архив» — через 90 дней или вручную. Можно пропустить «Планируется» → «В работе» (решение Q5).

## План наполнения (для Аналитика Run 3)

- Полная таблица переходов: FROM → TO + условие + RBAC-роль.
- `CREATED → PLANNED`: начальник производства.
- `PLANNED → IN_PROGRESS`: мастер (если materials OK) или `→ WAITING_MATERIALS`.
- `WAITING_MATERIALS → IN_PROGRESS`: когда материалы поступили.
- `IN_PROGRESS → PARTIAL`: автоматически ≥50%.
- `PARTIAL → COMPLETED`: автоматически 100%.
- `* → CANCELLED`: начальник/директор (кроме CREATED и ARCHIVED).
- `* → ARCHIVED`: авто через 90 дней или админ.
- `PLANNED → IN_PROGRESS` напрямую (Q5: «Взять в работу сразу»).
