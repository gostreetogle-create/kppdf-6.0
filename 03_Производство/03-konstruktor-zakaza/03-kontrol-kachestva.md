## Назначение

QA-чеклист и контроль качества в производстве. Проверка готовых изделий перед передачей на склад или клиенту.

## Контекст

Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §5.1: все позиции копируются из КП как snapshot. Качество проверяется до перехода в «Завершён». Если обнаружен брак → cross-ref на «Возврат/брак» (`03-vozvrat-i-brak.md`). Связь со Складом: только `ITEM` создаёт StockMovement при завершении.

## План наполнения (для Аналитика Run 3)

- Чеклисты по типам задач: из сборка, монтаж, работа.
- Кто проверяет: мастер / начальник /独立 QC (если есть).
- Критерии брака → переход в `03-vozvrat-i-brak.md`.
- Интеграция с `quantityActual`: фактКоличество может быть < plan из-за брака.
- Cross-ref на WriteOffAct (Склад) для списания брака.
