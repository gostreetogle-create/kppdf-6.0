# 03_Производство/03-konstruktor-zakaza/03-upakovka-i-dostavka.md — Упаковка и доставка

> ⚠️ **STUB.** Создан декомпозицией модуля `03_Производство/` (см. [MODULE-DECOMPOSITION-PLAN.md §3](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)).
> Контент будет наполнен Аналитиком Run 3 (ТЗ-008).

## Назначение

Упаковка готовых изделий и доставка клиенту. Cross-ref на модуль Склад (StockMovement: Приход при завершении ЗК, Shipment при отгрузке).

## Контекст

Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §6: при переходе ЗК в «Завершён» автоматически создаётся `StockMovement: type='IN', reason='PRODUCTION'` для каждой задачи-товара. Услуги/работы в склад НЕ приходят (МОДУЛЬ-СКЛАД-ПОДРОБНЫЙ §3 правило 6). Shipment (отгрузка клиенту) — отдельная операция Склада.

## План наполнения (для Аналитика Run 3)

- Автоматический приход на Склад при «Завершён»: `StockMovement.type='IN', reason='PRODUCTION'`.
- Только для `Product.kind='ITEM'`: услуги/работы/монтаж — НЕ приходят.
- Shipment (отгрузка клиенту) — через модуль Склад, не здесь.
- Упаковка: в MVP — нет отдельной операции, но упоминается в сценарии.
- Планирование «ожидаемой даты прихода» через `StockMovement.expectedAt`.
