## 0. Контекст

Производственный заказ (`ProductionOrder`) — «мостик» между Клиентом (деньги заплатил) и Производством (пора делать). Префикс `ЗК-XXXX`. Авто-создаётся при `Proposal.status='PAID'`. 8 статусов: `CREATED → PLANNING → AWAITING_MATERIALS → IN_PROGRESS → PARTIAL → COMPLETED → CANCELLED → ARCHIVED`. Каждый ЗК → несколько `ProductionTask` (по позициям КП). Задачи типа ITEM → автоматический `StockMovement:IN` при завершении (R1/GAP-009). Задачи SERVICE/WORK/INSTALLATION → только прогресс, без StockMovement.

5 папок модуля: `00-spr/` → `01-shablon/` → `02-redaktor-zakaza/` (UI: доска + детали ЗК + карточка задачи) → `03-zhiznennyj-cikl/` → `04-pravila/`. Конструктор-папка названа по **Стратегии C** (по URL/воркспейсам).
