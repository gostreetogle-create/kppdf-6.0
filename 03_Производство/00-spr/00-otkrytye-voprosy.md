## 1. Baseline OQ (5 шт)

| # | Тема | Тип | Сквозная связь | Статус |
|---|---|---|---|---|
| **OQ-001** | ЗК отменён (`ProductionOrder.status='CANCELLED'`) ПОСЛЕ оплаты. Как сигнал доходит до `Refund` в Модуле Финансы? | cross-module | → `CROSS-MODULE-OQ.md` §1 (Refund) | ⚠️ базовое (для Аналитика Run 1) |
| **OQ-002** | Если ЗК выполнен ЧАСТИЧНО (`ProductionOrder.status='PARTIAL'`), можно ли сделать частичную отгрузку (`Shipment`) на выполненную часть? | правило | — | ⚠️ базовое |
| **OQ-003** | Как кладовщик понимает, что 5 ЗК требуют одного и того же материала (конфликт резервов)? Создавать PurchaseRequest автоматически по `minStock`? | Склад | → `CROSS-MODULE-OQ.md` §2 (Reserve) | ⚠️ базовое |
| **OQ-004** | Если задача — «монтаж» (`Product.kind='INSTALLATION'`), нужен ли для неё отдельный акт выполненных работ, или достаточно ЗК = COMPLETED? | правило | — | ⚠️ базовое |
| **OQ-005** | Может ли кладовщик блокировать ЗК по причине обнаруженного брака материалов? Или только менеджер/admin может перевести ЗК в `AWAITING_MATERIALS`/`CANCELLED`? | RBAC | → RUN1 RBAC | ⚠️ базовое |

## 2. Направление ответов для Аналитика

| OQ | Рекомендуемая гипотеза (не финальная — обоснование в Run1 Аналитика) |
|---|---|
| OQ-001 | Сигнал через UI-флоу: начальник производства отменяет ЗК → видит «Бухгалтер, оформите Refund» (СПОР-12 ✅). `Refund` — отдельная сущность, не отрицательный Payment. |
| OQ-002 | Да, частичная отгрузка разрешена (V-019 ✅ через статусы Shipment `packed`/`shipped`/`delivered`/`partial`). |
| OQ-003 | Авто-`PurchaseRequest` при `availableQty < minStock` (V-018 ✅ зафиксировано). Конфликт резервов решается очередью. |
| OQ-004 | Достаточно ЗК = COMPLETED для service/work (V-015 ✅ все типы в ЗК, но StockMovement только для ITEM). |
| OQ-005 | Только начальник производства / директор для отмены. Кладовщик может ПОМЕТИТЬ проблему (signal), но не блокировать. |
