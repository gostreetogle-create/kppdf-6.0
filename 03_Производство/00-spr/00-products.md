# 03_Производство/00-spr/00-products.md — Продукты производства

> ⚠️ **STUB.** Создан декомпозицией модуля `03_Производство/` (см. [MODULE-DECOMPOSITION-PLAN.md §3](../../99_Справочники/MODULE-DECOMPOSITION-PLAN.md)).
> Контент будет наполнен Аналитиком Run 3 (ТЗ-008).

## Назначение

Продукты производства: товары (ITEM), услуги (SERVICE), работы (WORK), монтаж (INSTALLATION). Все типы из КП попадают в ЗК как задачи (СПОР-015, решение V-015: вариант A).

## Контекст

Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §5.2: одна позиция КП может породить одну или несколько задач. «Монтаж» (тип `Installation`) → отдельная задача в цехе «Выездная бригада». Тип «Работа» → задача в соответствующем цехе (например, «Конструкторский отдел»). Услуги/работы после выполнения фиксируются как `done`, но НЕ создают StockMovement.

## План наполнения (для Аналитика Run 3)

- Маппинг типов продукта на типы задач: `ITEM` → задача-изготовление, `SERVICE` → задача-услуга, `WORK` → задача-работа, `INSTALLATION` → задача-монтаж.
- Snapshot-семантика: `productSku`, `productName` фиксируются в момент создания ЗК.
- Связь с Складом: только `ITEM` создаёт StockMovement при завершении.
- Cross-ref на `Product.kind` из SCHEMA-CONSOLIDATED.
