## Назначение

Инварианты и валидации модуля Производство: ≥20 бизнес-правил. Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §5.

## Контекст

Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §5.1-§5.4: правила создания ЗК (4 правила), задач (7 правил), переходов (3 правила), цехов (3 правила). Всё ≥20 инвариантов.

## План наполнения (для Аналитика Run 3)

- **Создание ЗК:** Триггер = КП «Оплачено», один КП = один ЗК, snapshot позиций, авто-статус «Создан».
- **Задачи:** Один КП → одна/несколько задач, монтаж отдельно, частичная готовность через `quantityActual`.
- **Переходы:** «Завершён» = все задачи выполнены, «Отменён» = только начальник/директор, «Архив» = 90 дней.
- **Цеха:** Фиксированный справочник, задача к одному цеху, мастер может в нескольких.
- **Cross-module:** Завершён → StockMovement: Приход (только ITEM), отменён → Refund (СПОР-12).
- **Инварианты FK:** `parentProposalId` RESTRICT, `parentContractId` nullable.
- **Валидации:** plannedEndDate ≥ plannedStartDate, quantityActual ≤ quantityPlanned.
