> Контент будет наполнен Аналитиком Run 3 (ТЗ-008).

## Назначение

RBAC-матрица для модуля Производство: ≥35 правил, 7 ролей. Из [`МОДУЛЬ-ПРОИЗВОДСТВО.md`](../МОДУЛЬ-ПРОИЗВОДСТВО.md) §8.

## Роли (7)

| Роль | Описание |
|---|---|
| `admin` | Администратор системы |
| `manager` | Менеджер по продажам |
| `production` | Начальник производства |
| `production-master` | Мастер цеха |
| `storekeeper` | Кладовщик |
| `accountant` | Бухгалтер |
| `viewer` | Наблюдатель |

## Ключевые ограничения (из §8)

- Создание ЗК — ТОЛЬКО автоматически из КП «Оплачено» (никто не создаёт вручную).
- «Отменён» — ТОЛЬКО начальник производства или директор.
- «Архив» — ручной: админ; автоматический: через 90 дней.
- Мастер может работать только с назначенными ему задачами.

## План наполнения (для Аналитика Run 3)

- Полная матрица: Действие × Роль → ✅/❌ + условие (≥35 правил).
- Действия: Видеть доску, Взять в работу, Распределить, Назначить ответственного, Начать задачу, Отметить частично/готово, Отменить, Архивировать, Получать StockMovement.
- Условные ограничения: «свои» ЗК для менеджера, «назначенные» для мастера.
- Cross-ref на `RBAC-MATRIX.md` (глобальная матрица).
