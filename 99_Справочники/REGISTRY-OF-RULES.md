## 0. Контекст и метод

### 0.1 Принципы покрытия

1. Каждое правило извлечено из существующих МОДУЛЬ-документов и сводных справочников. Новых правил «от себя» НЕТ.
2. Каждое правило имеет машиночитаемый ID формата `RULE-{MODULE}-{TYPE}-{NNN}`.
3. Каждое правило ссылается на конкретный файл + раздел источника.
4. Каждое правило указывает следствие при нарушении (текст ошибки / поведение UI).
5. Противоречия между модулями отсутствуют (все Q и СПОР закрыты PO 24.06.2026).
