## §1. Общий подход (mirror КП)

### 1.1 Folder hierarchy (ОБЩАЯ ДЛЯ ВСЕХ 5 МОДУЛЕЙ)

```
0X_<Модуль>/
├── README.md                          # Точка входа модуля
├── 00-spr/                            # Справочники (фундамент)
│   ├── 00-README.md                   # Folder entrypoint (STUB)
│   ├── 00-glossary.md                 # Термины модуля
│   ├── 00-orgs.md                     # Организации и роли
│   ├── 00-products.md                 # Товары / продукты модуля
│   ├── 00-clients.md                  # Клиенты (если применимо)
│   ├── 00-out-of-scope.md             # Что НЕ входит в модуль
│   └── 00-otkrytye-voprosy.md         # OQ модуля (5 baseline)
├── 0X-shablon/                        # Шаблоны / скелеты (если применимо)
│   ├── 00-README.md
│   └── 0X-konstruktor.md              # Конструктор шаблонов
├── 0X-konstruktor-<X>/                # Сборка (UI + механика)
│   ├── 00-README.md
│   ├── 0X-*.md                        # Бизнес-логика экранов
├── 0X-zhiznennyj-cikl/                # Процесс / состояния
│   ├── 00-README.md
│   └── 0X-*.md                        # State-машина, переходы, конвертации
├── 0X-pravila/                        # Ограничения
│   ├── 00-README.md
│   ├── 0X-rbac.md                     # RBAC для ролей модуля
│   └── 0X-biznes-pravila.md           # Инварианты и валидации
```

### 1.2 Принципы (по AGENT-METHOD §3 Правило 3.1)

1. **Декомпозиция по 5 папкам** (00-spr → 01-shablon → 02-konstruktor-* → 03-zhiznennyj-cikl → 04-pravila) — структурная, не бизнес-классификация
2. **Каждый файл target 80-250 строк** (hard limit 400)
3. **Содержимое НЕ создаётся АРХИТЕКТОРОМ** — только мета (имя + 1-2 строки назначения). Содержание = Аналитик
4. **5 baseline OQ** для каждого модуля (если ещё нет)
5. **README файлы = folder entrypoints** (не трогаются после создания)

## §6. Cross-module dependencies

**Жёсткий DAG между модулями** (Phase 1 Bootstrap → Phase 2 UI order):

```
[01 КП] ──┐
          ├──→ [02 Договор] ──┐
                              ├──→ [03 Производство] ──┐
                                                      ├──→ [04 Склад] ──→ [05 Финансы]
```

### 6.1 Critical cross-references

| Контракт | Откуда | Куда | Когда стартует |
|---|---|---|---|
| КП→Договор | КП-А.КП «PAID» | Договор-А.Конвертация | После Run 1 КП (ТЗ-002) |
| Договор→Производство | Договор «SIGNED» | Производство-А.Создание ЗП | После Run 2 КП + Run 1 Договор |
| Производство→Склад | Производство «DONE» | Склад-А.Приход | После Run 2 КП + Run 2 Договор |
| Склад→Финансы | Склад «Расход» | Финансы-А.Платеж | После Run 3 КП + Run 3 Договор |
| Финансы→Все | «Закрыт период» | Аналитика / Reports | По завершении |
