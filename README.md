# Версия 6 — что внутри

> Этот файл — короткий указатель по папке `version_6/`. Открой нужный файл и работай с ним.

## Назначение

Здесь живут документы, которые описывают **как должна работать программа** с точки зрения бизнеса и удобства для пользователя.

Документы написаны **простым языком**, без технических терминов. Если появятся непонятные места или нестыковки — они собраны в файле `ОТКРЫТЫЕ-ВОПРОСЫ.md` и требуют решения.

## Для кого это всё

- **Менеджер по продажам** — главный пользователь системы. Все описания «как это работает» сделаны с точки зрения того, что он увидит на экране.
- **Ты (продакт-оунер)** — принимаешь решения по спорным вопросам, контролируешь что в `ОТКРЫТЫЕ-ВОПРОСЫ.md` отмечено как «решено».
- **Программист** — когда начнём кодить, он будет читать `МОДУЛЬ-КОММЕРЧЕСКОЕ-ПРЕДЛОЖЕНИЕ.md` как техническое задание (но **переведённое** на язык, понятный человеку). Технические детали реализации (какая база данных, какой фреймворк) **намеренно НЕ включены сюда** — они появятся когда мы определимся с технологией.

## Главная ценность модуля

Сейчас главное — доделать модуль «Коммерческое предложение» целиком и хорошо, прежде чем браться за что-то ещё. В шестой попытке это наш первый приоритет. Остальные модули (каталог товаров, склады, производство, финансы) — потом, по одному, без спешки.

## Как читать дальше

1. Открой `МОДУЛЬ-КОММЕРЧЕСКОЕ-ПРЕДЛОЖЕНИЕ.md` — это ядро. Прочитай от начала до конца один раз. Если что-то непонятно — посмотри в `ОТКРЫТЫЕ-ВОПРОСЫ.md`.
2. Зайди в `ОТКРЫТЫЕ-ВОПРОСЫ.md` — пройдись по каждому пункту, поставь свой ответ или задай уточняющий вопрос.
3. После того как по всем открытым вопросам будет ответ — документация готова. Можно начинать программировать.

## Важно

- Эти документы **заменяют** то, что было в старой папке `tech_spec/`. Та папка была перегружена техническими терминами, которые тебе мешали. Здесь — наоборот, никаких технических подробностей.
- Если программист в будущем попросит «техническое задание с деталями реализации», мы потом создадим отдельный файл в `archive/` или совсем в другом месте, где будут технические детали. Сейчас — этого делать не нужно, мы ещё не выбрали технологию.
