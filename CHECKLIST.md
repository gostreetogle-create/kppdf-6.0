## 0. Самообслуживание чек-листа

**Обновлять** если изменилась глобальная маршрутизация (новая роль, новый документ-источник, новый pipeline run, новый модуль).

**НЕ обновлять** для точечных правок — обновите источник по ссылке. Три исключения:
1. Изменился один STUB в `0X_*/` → править сам STUB.
2. Закрылась одна OQ → править `99_Справочники/OPEN-QUESTIONS-MASTER.md`.

## 6. Протокол возобновления (потеря контекста)

**Если контекст утерян (обрыв интернета / новая сессия / краш):**

1. **Открой `CHECKLIST.md`** (этот файл — точка входа для ИИ).
2. **Прочитай `PROJECT-STATE-LOG.md`** — последние 5 PSL сверху вниз. Определи последний завершённый шаг.
3. **Открой `MASTER-VISION.md`** — посмотри §4 «В работе параллельно» для текущего контекста.
4. **Спроси PO:** *«Контекст восстановлен. Последний шаг = **PSL-NNN — <название>**. Продолжить / новая задача / закрыть run?»*
5. **Если «продолжить»:** открой соответствующий `LAUNCH-X.md` (§4) и следуй §1 «Файлы для attach».
6. **Если задача сделана (PSL-XXX = DONE):** следующий run / новая задача — без повторов.
